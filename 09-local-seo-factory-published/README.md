# Фабрика локальных SEO-страниц / Local SEO Page Factory (опубликовано в marketplace)

> 📢 Опубликован в ASCN Marketplace: [RU](https://ascn.ai/ru/templates/local-seo-landing-page-generator-agent) · [EN](https://ascn.ai/en/templates/local-seo-landing-page-generator-agent)

Для локальных сервисных бизнесов: десятки уникальных страниц «услуга + город/район» с локальными деталями, FAQ и schema-разметкой — не дубли с заменой топонима.

## Ценность и ROI

SEO-подрядчик берёт за пакет из 30–50 локальных страниц $1000–5000 и делает его месяц. Скилл seo-page-builder — максимальный измеренный прирост в библиотеке cybos (+33pp): быстрая публикация 50+ страниц ценнее bespoke-текстов. Агент выдаёт 5 уникальных страниц за прогон, 20+ в месяц на автопилоте. При подписке $29/мес окупаемость 30–170x на одном проекте.

## Что делает

1. Строит матрицу «услуга x район/город» в Google Sheets: все комбинации, приоритет каждой по спросу (оценка через веб-поиск: конкуренция в выдаче, наличие агрегаторов, локальные сигналы).
2. Генерирует страницы по утверждённой матрице: каркас из knowledge/PAGE_TEMPLATE.md, локальные детали района по методике knowledge/LOCAL_SIGNALS.md (ориентиры, транспорт, специфика застройки — с источниками).
3. Каждая страница уникальна по содержанию: свой первый абзац с прямым ответом, свои локальные факты, свой FAQ 5 вопросов, черновик schema LocalBusiness/Service.
4. Выход — пакет: Google Docs (по документу на страницу или сборник) + HTML-файлы в Drive + инструкция по публикации (URL-структура, title/description, перелинковка).
5. По вторникам — очередной батч из 5 страниц по матрице (после утверждения плана), статусы в матрице обновляются.
6. Проверка на каннибализацию и дубли: страницы одной услуги в соседних районах различаются фактурой, а не синонимами.

## Состав папки

- `workspace/` — копируется в воркспейс агента ASCN (IDENTITY, SOUL, ONBOARDING, скиллы page-matrix и page-generator, справочники PAGE_TEMPLATE и LOCAL_SIGNALS).
- `DEPLOY_PROMPT.md` — промпт для Claude Code, который переносит всё в ASCN.
- `INTEGRATIONS.md`, `AUTOMATIONS.md` — справка по интеграциям и готовый текст cron-задачи.

## Быстрый деплой

1. Откройте Claude Code в этой папке.
2. Вставьте содержимое `DEPLOY_PROMPT.md`.
3. Напишите агенту «Привет, начнём настройку» — дальше он сам проведёт онбординг (бизнес, услуги, гео, шаблон, тестовая страница).

## Требуемые интеграции

Google Sheets (обязательно), Google Docs (обязательно), Google Drive (обязательно). Google Search Console (рекомендуется). Плюс встроенный веб-поиск.

## Ограничения (сознательные)

- Автопубликации в CMS нет: у платформы нет браузерной автоматизации, коннекторов к WordPress/Tilda/Bitrix нет. Выход — готовый пакет (Docs + HTML в Drive) и пошаговая инструкция; публикует пользователь или контент-менеджер за 5–10 минут на страницу. Продвинутый вариант — кастомный MCP к API вашей CMS (WordPress REST и т.п.), тогда публикацию можно поручить агенту.
- Данные о точном поисковом спросе (Wordstat/Ahrefs) недоступны — приоритеты матрицы оцениваются по косвенным сигналам веб-поиска и данным Search Console (если подключена). Это оценка, не измерение.
- Агент не выдумывает локальные факты: каждый ориентир и деталь района — из открытых источников с URL; не нашёл — блок становится нейтральным, без фейковой «локальности».
