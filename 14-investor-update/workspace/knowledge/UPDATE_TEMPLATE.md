# Шаблон инвестор-апдейта
Проверенная структура письма. Тон корректируется по прошлым письмам пользователя (лист archive); ниже — дефолт.

## Структура
Тема: <Компания> — Investor Update, <месяц год>
1. TL;DR — 3 строки: состояние одним предложением, главная цифра месяца, главный ask.
2. Metrics — таблица: метрика / прошлый месяц / этот месяц / MoM. 6–10 строк.
3. Highlights — 3–5 буллетов, каждый с цифрой.
4. Lowlights — 2–3 буллета: что пошло не так + что делаем. Обязательная секция.
5. Asks — 1–3 конкретных: кого/что/сколько и как помочь одним действием.
6. Follow-up — что вышло из прошлых asks, спасибо поимённо.
7. Подпись.

## Пример (сокращённый)
Тема: Acme — Investor Update, июнь 2026

TL;DR: Растём +9% MoM по MRR при runway 10.7 мес; готовим раунд на Q4; ищем интро к финтех-командам 50–200 человек.

Metrics: [таблица из скилла investor-update]

Highlights:
- MRR $19.9k (+9.3%): 5 новых клиентов, из них 2 — годовые контракты с предоплатой.
- Выпустили интеграцию с 1С: 14 входящих запросов за месяц, 3 конвертировались в сделки.
- NRR 108%: апгрейды впервые перекрыли churn.

Lowlights:
- Потеряли клиента X ($800 MRR): ушёл к конкуренту из-за отсутствия SSO. SSO — в роадмапе Q3, спека готова.
- Найм senior backend буксует второй месяц: 47 кандидатов, 0 офферов. Пересмотрели вилку (+15%), переоткрыли JD.

Asks:
1. Интро к финтех-компаниям 50–200 человек — наш ICP; особенно ценны портфельные.
2. Форвардните JD senior backend [ссылка] — вилка обновлена.

Follow-up: Спасибо Николаю за интро к Y — дошло до оплачиваемого пилота. Обещали дашборд self-serve — выпустили 12.06.

— Иван, CEO Acme

## Тон (дефолт, если archive пуст)
- Деловой, короткие фразы, без хайпа и превосходных степеней.
- Каждый тезис с цифрой: «значительно выросли» запрещено, только «+9.3%».
- Плохие новости — прямо и с планом действий: инвесторы уважают контроль, а не идеальность.
