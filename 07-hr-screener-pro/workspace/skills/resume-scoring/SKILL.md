# Скоринг резюме

Цель: балл A/B/C с полной трассировкой — по каждому критерию видно, что найдено в резюме и почему такой балл. Проверяемость важнее скорости.

## Алгоритм
1. Извлеки из PDF/DOCX/текста: ФИО, контакты, роли с датами (месяц-год), компании, обязанности и достижения, стек/навыки, образование, языки, ссылки (GitHub, портфолио). PDF-скан не читается → флаг need_manual, строка в таблице со статусом new, пометка пользователю.
2. Определи вакансию: тема письма, текст, папка. Неоднозначно → спроси, не скорь наугад.
3. Скоринг по рубрике вакансии (knowledge/VACANCY_RUBRIC.md):
   - must-have: каждый критерий 0 / 1 / 2 (нет подтверждения / частично / уверенно, с фактом). Любой must-have = 0 → итог не выше C, причина фиксируется в трассировке.
   - nice-to-have: 0/1/2, умноженные на вес.
   - Итог = взвешенная сумма в % от максимума. Пороги из рубрики (дефолт: A ≥ 75%, B 55–74%, C < 55%).
4. Флаги (пометки «проверить на интервью», не приговор):
   - ai_style: однотипные буллеты одинаковой длины во всех местах работы + плотность штампов («результативный командный игрок», «leveraged», «spearheaded») без единой цифры и названия проекта;
   - date_conflict: пересекающиеся полные занятости; date_gap: разрыв >6 месяцев без объяснения; job_hopping: 3+ подряд мест по <12 месяцев (контрактную работу не считать — смотри контекст);
   - copy_paste: обязанности дословно повторяют текст нашей вакансии.
5. Запиши в Sheets: score, флаги, resume_link, status=screened; полную трассировку — на вкладку «trace» (или в Docs, если подключён). A → алерт сразу; B/C → в пятничную сводку.

## Выход (трассировка)
```
Кандидат: Анна Крылова — Middle QA — балл B (68%)
| Критерий (вес) | Найдено в резюме | Балл |
|---|---|---|
| must: автотесты Python (3) | «2 года pytest + Selenium в Ozon», стр. 1 | 2 |
| must: API-тестирование (3) | Postman в стеке, задач не описано | 1 |
| must: SQL (2) | «сложные выборки PostgreSQL для тестовых данных» | 2 |
| nice: CI/CD (2) | GitLab CI упомянут в стеке | 1 |
| nice: нагрузочное (1) | не найдено | 0 |
Флаги: date_gap (окт 2023 — май 2024 без записей) — проверить на интервью.
Итог: 68% → B. Кандидат в шорт-лист при нехватке A.
```

## Краевые случаи
- Дубликат отклика (совпадает email/телефон) → строку не создавай; обнови существующую, в notes «повторный отклик <дата>».
- Резюме на английском при русской вакансии (или наоборот) → скорь как обычно; язык влияет на балл, только если он критерий рубрики.
- Сопроводительное письмо без резюме → черновик вежливого запроса резюме (на утверждение), статус need_cv.
- 50+ резюме одной пачкой → скорь всех, в чат — топ-10 и распределение A/B/C, остальное в таблицу без пересказа.
