# Scorecard интервью

> Скилл — одна повторяемая задача, описанная как рецепт. Шаги, лимиты и форматы можно править обычными словами.

Цель: из сырого транскрипта — структурированная оценка по компетенциям с цитатами-доказательствами и рекомендация hire/no-hire. Ни одной оценки без цитаты.

## Алгоритм
1. Определи кандидата и вакансию: имя файла («<кандидат> — <роль>»), содержимое; неоднозначно — спроси. Подними строку из Sheets и трассировку скоринга резюме.
2. Возьми компетенции из рубрики вакансии (knowledge/VACANCY_RUBRIC.md, раздел «Компетенции для интервью») + постоянные: коммуникация, мотивация.
3. По каждой компетенции найди относящиеся фрагменты и оцени 1–4:
   1 — не смог показать / уходил от ответа; 2 — поверхностно, без деталей; 3 — уверенно, с конкретикой (цифры, решения, своя роль); 4 — глубина + рефлексия (что пошло не так, что менял бы).
   К каждой оценке — 1–2 дословные цитаты. Компетенцию не спрашивали → «не проверено» (не 1) и в список «вопросы к следующему этапу».
4. Red flags, каждый с цитатой: противоречия с резюме (даты, роль, масштаб); присвоение командных результатов («мы сделали» → на уточнении не может назвать свой вклад); негатив о прошлых работодателях; уклонение от прямого вопроса 2+ раза.
5. Сверь с флагами скоринга резюме: ai_style, date_gap проверены на интервью? Зафиксируй результат (подтверждён / снят).
6. Рекомендация: hire (средняя ≥3 и нет red flags) / borderline (2.5–3 или один red flag — напиши, что уточнить) / no-hire (<2.5 или ≥2 red flags). Это рекомендация — решение за пользователем.
7. Сохрани scorecard (Docs «Scorecard — <кандидат>» или текст в чат), ссылку и status=scorecard в Sheets.

## Выход
```
Scorecard: Анна Крылова — Middle QA — интервью 12.07, 55 мин
- Автотесты — 3 — «покрыла регресс на 70%, гоняли в ночном CI, падения разбирала по утрам»
- API — 2 — «в основном руками через Postman... автоматизировать не успевали»
- SQL — не проверено — — (в вопросы следующего этапа)
- Коммуникация — 3 — отвечает по сути, уточняет вопрос перед ответом
Red flags: нет.
Флаг из резюме date_gap: объяснён (декрет) — снят.
Рекомендация: borderline → технический этап с фокусом на API-автоматизацию и SQL. Вопросы: 3 шт, приложены.
```

## Краевые случаи
- Транскрипт без разметки спикеров → раздели по контексту (интервьюер задаёт вопросы); не уверен — предупреди, что цитаты могут смешивать спикеров.
- Короткий/обрывочный транскрипт (<15 минут разговора) → scorecard с пометкой «данных мало, оценки предварительные».
- Два кандидата в одном файле → раздели и сделай два scorecard; два интервьюера — нормально, помечай, кто спрашивал, если различимо.
- Кандидата нет в таблице → создай строку со статусом interview и спроси, откуда кандидат (источник).
