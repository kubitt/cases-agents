# Схема реестра счетов (Google Sheets «Реестр счетов»)

> Ваш справочный файл. Агент читает его перед работой и дополняет по ходу. Правьте свободно.

Лист `invoices` — источник правды. Одна строка = один счёт.

- invoice_id — текст — номер счёта, уникален
- client — текст — юрлицо / имя клиента
- contact_name — текст — к кому обращаемся в письме
- email — текст — адрес для напоминаний; несколько — через запятую, первый основной
- amount — число — сумма к оплате (при частичной оплате — остаток)
- currency — текст — RUB / USD / EUR …
- issued_date — дата — когда выставлен
- due_date — дата — срок оплаты
- days_overdue — число — сегодня − due_date; агент обновляет ежедневно
- stage — текст — S0–S4
- status — текст — current / overdue / promised / paid_claimed / paid / disputed / escalated / written_off
- promise_date — дата — обещанная дата оплаты
- promise_log — текст — «05.07: до 10.07 → сорвано; 11.07: до 15.07 → ждём»
- dispute_reason — текст — суть спора одной фразой
- vip — да/пусто — не трогать без команды пользователя
- last_touch — дата — последнее отправленное касание
- next_touch — дата — ближайшее плановое касание
- thread_log — текст — «дата — событие», через «;», новые сверху
- notes — текст — частичная оплата, договорённости, прочее

## Правила ведения
- paid ставит только пользователь: командой «оплачен счёт N» или правкой ячейки. paid_claimed — заявление должника, ждёт подтверждения.
- Пополнение реестра: строка вручную, CSV-файл в чат или Drive («перенеси в реестр»), команда «добавь счёт: клиент, сумма, срок».
- Импорт CSV: агент маппит колонки по заголовкам, показывает предпросмотр 3 строк, пишет в таблицу только после «ок». Существующий invoice_id — обновить строку, не создавать дубль.
- Смена email клиента — фиксируется в notes с датой (важно для триггера Gmail).
- Лист `paid_archive` (опционально): агент переносит туда счета через 30 дней после оплаты, чтобы invoices не разрастался.
