# Триаж ответов должников

Цель: ни один ответ и ни одно обещание не теряются; статус в реестре всегда отражает реальность.

## Классы ответов
| Класс | Признаки | Действие |
|---|---|---|
| оплатил | «оплатили», платёжка во вложении | status=paid_claimed; попросить пользователя подтвердить поступление; paid — только после подтверждения |
| обещал | «оплатим до/в …» | извлечь дату → promise_date, status=promised, next_touch=promise_date+1, запись в promise_log |
| спорит | «не согласны», «акт не подписан», «претензия по качеству» | status=disputed, касания стоп, суть в dispute_reason, эскалация пользователю |
| просит рассрочку/перенос | «частями», «перенесите срок» | не решать самому: пользователю с проектом ответа |
| автоответ / не туда | отпуск, сотрудник уволился | сдвинуть next_touch на дату возврата / спросить у пользователя новый контакт |

«Молчит» — не класс письма, а состояние: нет ответа к next_touch → следующая стадия по dunning-sequence.

## Алгоритм
1. Найди счёт(а) по адресу отправителя. Адреса нет, но домен совпадает с клиентом — считай тем же клиентом, отметь новый контакт в notes. Совсем не нашёл — сообщи пользователю, реестр не трогай.
2. Классифицируй по таблице. Смешанный ответ («половину оплатим, по остальному вопрос») — веди по худшему классу, детали раздели в thread_log.
3. Обнови реестр: status, promise_date, dispute_reason, thread_log (формат: «дата — класс — цитата ≤15 слов»).
4. Обещание без даты («на днях», «скоро») → promise_date = сегодня+3; в ответе вежливо попроси точную дату.
5. Подготовь черновик ответа по DUNNING_PLAYBOOK.md, раздел «Ответы на ответы».
6. Пользователю: цитата письма + класс + черновик на утверждение. Спор или «оплатил» на сумму ≥ escalation_threshold — пометь «важное» первой строкой.

## Трекинг обещаний
- promise_log: записи вида «05.07: обещали до 10.07 → сорвано; 11.07: обещали до 15.07 → ждём».
- promise_date+1 без подтверждения оплаты → письмо-возврат: вежливо, со ссылкой на договорённость, просьба новой даты.
- 2 сорванных обещания подряд → пометка «хронический» в сводке, рекомендация эскалации; стадию не понижать.

## Краевые случаи
- Платёжка во вложении → всё равно paid_claimed: платёжка ≠ поступление денег.
- Агрессия или грубость → не зеркалить; черновик нейтральный, только факты; пользователю — пометка о тоне.
- Ответ про часть счетов («оплатили 214») → закрывай (после подтверждения) только названный; остальные живут по своим стадиям.
