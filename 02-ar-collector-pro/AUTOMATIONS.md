# Автоматизации: AR-коллектор PRO

## Cron-задачи

### 1. Ежедневный проход по реестру
- Cron: `0 11 * * *` (ежедневно, 11:00)
- Промпт задачи: «Открой Google Sheets "Реестр счетов", лист invoices. Шаг 1: обнови days_overdue = сегодня − due_date для всех счетов со статусом кроме paid/written_off. Шаг 2: проверь обещания — у кого status=promised и promise_date < сегодня, а оплата не подтверждена: верни статус overdue, допиши в promise_log "сорвано", подготовь письмо-возврат к обещанию по knowledge/DUNNING_PLAYBOOK.md. Шаг 3: для каждого счёта со статусом current/overdue определи стадию по скиллу dunning-sequence и, если наступил next_touch, подготовь черновик письма в Gmail; несколько счетов одного клиента объединяй в одно письмо. Если сегодня суббота или воскресенье — выполни только шаги 1–2, новые черновики напоминаний не готовь. Шаг 4: отправь сводку: черновиков готово (клиент — счёт — сумма — стадия), сорванные обещания, споры без ответа больше 2 дней, счета без email. Письма НЕ отправляй — только черновики на утверждение.»
- Канал: Telegram.

### 2. Недельный отчёт по дебиторке
- Cron: `0 16 * * 5` (пятница, 16:00)
- Промпт задачи: «По Google Sheets "Реестр счетов" собери недельный отчёт: 1) всего в работе — сумма и число счетов по корзинам старения 0–30 / 31–60 / 61–90 / 90+ дней; 2) собрано за неделю (status=paid с датой этой недели) — сумма и список; 3) обещания на следующую неделю (кто, сколько, когда); 4) споры (кто, суть из dispute_reason, сколько дней висит); 5) сорванные обещания за неделю; 6) топ-5 крупнейших просрочек с рекомендацией по каждой (продолжать касания / эскалация / решение пользователя — по правилам DUNNING_PLAYBOOK.md). Формат: не больше 25 строк, суммы с валютой. Отправь в Telegram.»
- Канал: Telegram.

## Триггеры

### 3. Ответ должника (Gmail: новое письмо)
- Условие: адрес отправителя есть в колонке email «Реестра счетов» и статус его счёта ∈ {current, overdue, promised, disputed, paid_claimed}.
- Действие: «Разбери письмо по скиллу reply-triage: классифицируй (оплатил / обещал / спорит / просит рассрочку / автоответ), обнови status, promise_date, promise_log и thread_log в реестре. Подготовь черновик ответа по DUNNING_PLAYBOOK.md (раздел "Ответы на ответы") и пришли в Telegram: цитата письма должника + класс + черновик на утверждение. Заявление об оплате не помечай paid — только paid_claimed до подтверждения пользователя.»
- Кредиты: триггер срабатывает на КАЖДОЕ входящее письмо ящика (для фильтрации). Если ящик общий, с большим потоком — выключите триггер: ответы разберёт ежедневный проход в 11:00 (шаг добавляется в промпт задачи №1: «проверь входящие от адресов реестра за сутки»).

Тихие часы 23:00–08:00: обе задачи и уведомления — вне этого окна.
