# Структура ICP-документа (Google Docs)

> Ваш справочный файл. Агент читает его перед работой и дополняет по ходу. Правьте свободно.

1. Главный вывод — одна фраза с цифрой. Датируется. Пример: «Наш клиент — стоматологии 2–5 филиалов, пришедшие по рефералу: 73% конверсия против 22% средней (данные 47 сделок, июль 2026)».
2. Ядро ICP: 2–3 сегмента. По каждому: определение (ниша, размер, гео) → win rate и N → средний чек и цикл → почему выигрываем (из данных).
3. Анти-профиль: кого НЕ берём. Сегменты с конверсией ниже средней в 2+ раза, с расчётом цены: «enterprise: 12% конверсии, 40% времени команды».
4. Пограничные сегменты: данных мало, но сигнал интересный — что проверить (N сделок, гипотеза).
5. Персоны (по сегментам ядра): роль → боли цитатами → триггеры → возражения и ответы → чек-лист скоринга → где искать.
6. Причины проигрышей: топ-3 категории с долями и типовыми формулировками.
7. Проверить интервью: гипотезы, не подтверждаемые данными.
8. Changelog: дата → что изменилось в ICP → на каких данных (заполняется при каждом пересчёте).

Правило: ни одного утверждения без (N=…, %) или пометки «гипотеза».
