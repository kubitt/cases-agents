# Месячное закрытие

> Скилл — одна повторяемая задача, описанная как рецепт. Шаги, лимиты и форматы можно править обычными словами.

Цель: из сырых выгрузок — пакет, который не стыдно показать бухгалтеру и инвестору: P&L, движение денег, комментарии к отклонениям, одностраничник.

## Алгоритм
1. Источники: файлы за отчётный месяц из Drive-папки «Выгрузки» (CSV/XLSX банковской выписки, выгрузка продаж) + Sheets «Финансы» (если ведётся вручную). Список ожидаемых файлов — в knowledge/CHART_OF_ACCOUNTS.md, раздел «Источники».
2. Категоризация: каждую операцию отнеси к категории по правилам CHART_OF_ACCOUNTS.md (маппинг «назначение платежа/контрагент → категория»). Не распознал — в категорию «Разобрать» и списком на уточнение пользователю (не более 10 крупнейших, остальное — суммой).
3. Посчитай: выручка по направлениям, расходы по категориям, EBITDA-приближение, денежный поток (поступления/списания/остаток на конец), дельты к прошлому месяцу и к среднему за 3 месяца.
4. Сверка: остаток на конец = остаток на начало + поступления − списания. Не сходится — покажи расхождение суммой и списком подозрительных операций. НЕ подгоняй.
5. Собери пакет в Google Sheets «Финпакет <YYYY-MM>»: листы P&L, CashFlow, Разобрать. Одностраничник для основателя — по knowledge/REPORT_TEMPLATE.md в Google Docs.
6. В чат/Telegram — резюме 10 строк: выручка, расходы, дельта, топ-3 отклонения с причинами, остаток, runway по текущему burn.

## Краевые случаи
- Нет выгрузки за месяц → напомни пользователю список недостающих файлов, собери частичный пакет с пометкой «неполные данные».
- Дубли операций (банк + касса) → правило дедупа из CHART_OF_ACCOUNTS.md; сомнение — в «Разобрать».
- Валюты: конвертируй по курсу на дату операции (веб-поиск ЦБ/ЕЦБ), курс фиксируй в отдельной колонке.
