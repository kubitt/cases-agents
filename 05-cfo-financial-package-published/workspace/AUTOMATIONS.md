# Автоматизации: Финансовый пакет + runway (аутсорс-CFO)

> Файл живёт в воркспейсе агента. Агент читает его при создании и изменении cron-задач. Расписания и формулировки можно менять — агент будет следовать обновлённой версии.

## Триггеры

### 1. Новая выгрузка (Google Drive: новый/изменённый файл)
- Условие: файл в папке «Выгрузки», расширение csv/xlsx/xls. Файлы, созданные агентом, и папку «Финансы/Отчёты» — пропускать.
- Действие: «Определи тип файла (банковская выписка / CRM-выгрузка / прочая таблица) по колонкам и имени. Валидация: период данных, обязательные колонки (дата, сумма, назначение/контрагент), пересечение с уже загруженными периодами (дубли). Классифицируй операции по knowledge/CHART_OF_ACCOUNTS.md и добавь в Sheets "Финансы — операции", лист transactions, с колонкой source_file. Операции без правила классификации: суммой ≥ порога из памяти — собери в список вопросов, меньше порога — категория "прочее" с пометкой uncategorized. Если файл — выписка с исходящим остатком: обнови лист balances (счёт, остаток, дата). Ответь: файл, период, строк принято, дублей пропущено, вопросы по классификации (≤10 за раз). Ничего не удаляй; при подозрении на повторную загрузку того же периода — спроси, не пиши молча.»
- Кредиты: триггер срабатывает на каждый файл в папке — кладите туда только выгрузки. Массовая загрузка истории (10+ файлов) — попросите агента разобрать пакет одной командой при выключенном триггере.

## Cron-задачи

### 2. Месячный финансовый пакет
- Cron: `0 10 3 * *` (3-е число, 10:00)
- Промпт задачи: «Собери месячный финансовый пакет за прошлый календарный месяц по скиллу month-close: проверка полноты данных, P&L, движение денег со сверкой остатков, комментарии к отклонениям, одностраничник по knowledge/REPORT_TEMPLATE.md. Если выгрузки за месяц неполные (дыра в датах, нет файла по одному из счетов) — пакет не считай молча: перечисли, чего не хватает, и запроси файлы. Готовый пакет: таблицы в Sheets "Финансы — операции" (листы pnl, cashflow), нарратив в Docs "Финпакет <YYYY-MM>" в папке "Финансы/Отчёты", одностраничник текстом в канал со ссылками.»
- Канал: Telegram.
- Примечание: дата по умолчанию — 3-е число; если бухгалтерия закрывает месяц позже, онбординг сдвигает cron.

### 3. Обновление runway
- Cron: `0 9 * * 1` (понедельник, 09:00)
- Промпт задачи: «Обнови 13-недельный прогноз кэша по скиллу runway-forecast. Week-0 = последний подтверждённый остаток из листа balances; если остаток старше 7 дней — начни сообщение с запроса свежей выписки или остатка сообщением и пометь прогноз "по данным на <дата>". Посчитай 3 сценария (база/оптимист/пессимист), запиши в лист runway, отправь сводку ≤12 строк: остаток и дата, runway по сценариям, минимальная точка, изменение за неделю и его причина, ближайшие крупные оттоки. Если runway в базе < 12 недель или в пессимисте разрыв в ближайшие 4 недели — первой строкой ВНИМАНИЕ.»
- Канал: Telegram.

Тихие часы 23:00–08:00: задачи в 09:00 и 10:00 — вне окна.
