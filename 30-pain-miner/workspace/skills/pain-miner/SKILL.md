# pain-miner

## Когда использовать
Когда пользователь говорит «запусти разведку болей», «найди боли», «обнови инсайты», «pain mining», или когда срабатывает задача по расписанию. Этот навык — полный конвейер: собрать обсуждения → вытащить боли → сгруппировать → собрать инсайты → отдать отчёт.

## Главный принцип
На выходе НЕ дамп постов, а структурированные инсайты: кластеры болей с частотностью, дословными цитатами и готовыми углами для лендингов и контента. Цитаты — только дословные. Ники не хранить.

## Инструменты под каждый этап
- Поиск: `clawman__web_search`.
- Чтение страниц и вызов JSON-API источников (Hacker News, Stack Exchange): `clawman__web_fetch` (GET по URL) или `shell` + curl.
- Кэш и артефакты: `file_write` / `file_read` / `file_list` (папка `files/painminer/`).
- Идемпотентность: `memory_store` / `memory_recall` (ключи `pain_miner_*`).
- Выгрузка (только по подтверждению): `clawman__integrations` (Google Sheets/Docs), `clawman__channels` (Telegram).
- Расписание: `clawman__cron`.

## Параметры (из USER.md, есть дефолты)
domain · topics[] · languages (ru, en) · since (6 мес) · sources (HN + Stack Exchange вкл; Reddit/web по наличию) · лимит на прогон (~20 обсуждений на тему) · веса 0.4/0.3/0.3.
Рецепты источников и словари маркеров — в `knowledge/PAIN_SOURCES.md`. Контекст домена — в `knowledge/DOMAIN.md`.

---

## Этап 1 — COLLECT (сбор)
Для каждой темы и каждого включённого источника:
1. Составь запросы = `seed_keywords` темы × маркеры боли (`knowledge/PAIN_SOURCES.md`, RU+EN).
2. Дёрни источник (рецепты — в `knowledge/PAIN_SOURCES.md`):
   - **Hacker News** (без ключа, делай первым): GET к Algolia API, разбери JSON.
   - **Stack Exchange** (без ключа): search/advanced по нужным сайтам.
   - **Reddit**: OAuth, если есть ключи в Секретах; иначе `web_search "<seed> site:reddit.com"` + чтение треда.
   - **Веб/форумы**: `web_search` → `web_fetch` 1–2 лучших страниц за реальными цитатами.
3. Нормализуй каждый элемент в **RawItem**. `lang` определи по тексту.
4. **Предфильтр ДО разметки** (экономия): оставляй кандидата, только если в тексте есть маркер боли ИЛИ вопросительная форма. Остальное не разбирай.
5. **Дедуп**: по `url` и по нормализованному хешу текста. Пропусти `id`, уже лежащие в `pain_miner_seen_ids` — повторный прогон не пережёвывает старое.
6. **Приватность**: автора не сохраняй — вместо ника `author_hash` (sha256). Запиши собранное в `files/painminer/raw/<source>.jsonl` (append).

```
RawItem = { id, source, url, author_hash, timestamp, title, text, engagement, lang, topic }
```

## Этап 2 — EXTRACT (извлечение боли)
Бери кандидатов из `files/painminer/raw/*.jsonl` (можно мысленно батчами по 10–20). Для каждого реши: выражает ли он проблему/боль/фрустрацию/неудовлетворённую потребность в нашем домене (см. `knowledge/DOMAIN.md`).
Если да — **PainItem**:
- `pain_statement`: 1 нейтральное предложение по-русски — суть боли.
- `verbatim`: короткая ДОСЛОВНАЯ цитата (≤25 слов), язык оригинала. Только дословно из текста — не сочинять.
- `job_to_be_done`: что человек на самом деле пытается сделать.
- `tools_mentioned`: упомянутые инструменты/конкуренты.
- `intensity`: 1–3 (сила фрустрации).
Если не боль — пропусти.
**Самопроверка:** каждая `verbatim` обязана быть дословной подстрокой исходного `text`. Не нашёл точную цитату — выкинь элемент (лучше меньше, но честно).
Запиши боли в `files/painminer/pains.jsonl`. Не переразбирай уже размеченные `id`.

```
PainItem = { raw_id, pain_statement, verbatim, job_to_be_done, tools_mentioned[], intensity, source, url, engagement, timestamp }
```

## Этап 3 — CLUSTER (группировка)
Сгруппируй все `pain_statement` в 6–12 тематических кластеров. Каждому — короткое русское название. Похожие боли — в один кластер. Для больших объёмов группируй по частям, затем объедини близкие кластеры. Запиши `files/painminer/clusters.json` (cluster_id → название + список raw_id).

## Этап 4 — BUILD (сборка инсайтов + ранжирование)
Для каждого кластера посчитай:
- `size` и долю от общего;
- **тренд**: разбей окно на бакеты по времени, оцени динамику — растёт / стабильно / разовая вспышка;
- разбивку по источникам;
- **топ-5 verbatim** по `intensity × engagement` — со ссылками;
- **язык аудитории**: частотные слова/биграммы по `verbatim` (убери стоп-слова RU+EN) — материал для SEO и заголовков;
- сводный `tools_mentioned`;
- **угол лендинга** (1–2 предложения языком аудитории) и **угол контента** (тема статьи/видео).
Ранжируй: `priority = norm(size)*0.4 + norm(recency)*0.3 + norm(engagement)*0.3` (веса из USER.md). Запиши `files/painminer/insights.json`.

## Этап 5 — REPORT (отчёт и выгрузка)
1. Сформируй `files/painminer/insights.md` (шаблон ниже), `insights.csv` и `insights.json` (одна строка/объект = один кластер).
2. В чат — краткая сводка: топ-5 кластеров (по 1 цитате + оба угла) и метрики прогона (собрано / размечено / сколько запросов).
3. **Выгрузка ТОЛЬКО по явному подтверждению** пользователя в этом разговоре:
   - Google Таблица — одна строка = один кластер (`clawman__integrations`);
   - Telegram-дайджест — топ-кластеры (`clawman__channels`).
   Без «да, выгружай» — только файлы и сводка в чат.

### Шаблон insights.md
```
# Инсайты болей — {дата}
Собрано: {N} обсуждений · Болей: {M} · Кластеров: {K} · Источники: {...}

## {ранг}. {Название кластера} · приоритет {0.00}
- Размер: {n} ({доля}%) · Тренд: {растёт/стабильно/вспышка}
- Источники: {HN n · Reddit n · ...}
- Цитаты:
  - «{verbatim}» — [{source}]({url})
  - «{verbatim}» — [{source}]({url})
- Язык аудитории: {слово, фраза, ...}
- Инструменты рядом: {...}
- 🎯 Угол лендинга: {...}
- ✍️ Угол контента: {...}
```

## Идемпотентность и затраты
- Перед прогоном — `memory_recall pain_miner_seen_ids`; после — допиши новые `id`.
- Лог на каждом этапе: собрано / отфильтровано / размечено / число запросов.
- Лимит на прогон по умолчанию ~20 обсуждений на тему (поправь в USER.md). Команда «полный прогон» снимает лимит.

## Если что-то сломалось
Источник упал → не падать целиком: пометь источник `failed`, продолжи с остальными, в сводке честно напиши, что не сработало. Это поведение надёжного агента — лучший безопасный частичный результат.

## Примеры (калибровка) — что считать болью и как формулировать
Ниже — реальные примеры из калибровочного прогона по Hacker News. Так выглядит хорошая разметка боли.

**Пример 1 — боль есть.**
- Текст: "I was tired of AI that just \"talks about\" work. I wanted AI that actually does things."
- pain_statement: Люди устали от ИИ, который только рассуждает о работе, а не выполняет её.
- verbatim: "AI that just \"talks about\" work"
- job_to_be_done: Получить ИИ, который сам делает задачи, а не описывает шаги.
- tools_mentioned: [ChatGPT, Claude]
- intensity: 3

**Пример 2 — боль есть.**
- Текст: "I got tired of spending 2+ hours a day in my inbox writing emails I'd already written a hundred times before."
- pain_statement: Человек тратит по 2+ часа в день на однотипные письма в почте.
- verbatim: "spending 2+ hours a day in my inbox writing emails I'd already written"
- job_to_be_done: Автоматизировать ответы на повторяющиеся письма.
- tools_mentioned: [Gmail]
- intensity: 3

**Пример 3 — боль есть.**
- Текст: "Sales reps waste hours copying data between CRMs and emails."
- pain_statement: Продавцы теряют часы на ручной перенос данных между CRM и почтой.
- verbatim: "Sales reps waste hours copying data between CRMs and emails"
- job_to_be_done: Убрать ручной ввод данных в CRM.
- tools_mentioned: [CRM, Salesforce, HubSpot]
- intensity: 2

**Пример 4 — боль есть (no-code).**
- Текст: "the automations are cumbersome to edit and very limited"
- pain_statement: Готовые автоматизации неудобно править и они слишком ограничены.
- verbatim: "the automations are cumbersome to edit and very limited"
- job_to_be_done: Собирать гибкую автоматизацию без хрупких воркфлоу.
- tools_mentioned: [Zapier, Make.com, n8n]
- intensity: 2

**Пример 5 — НЕ боль (отсекаем).** Новость о раунде/бенчмарке без личной фрустрации и без «мне нужно сделать X» → is_pain=false, пропускаем.

**Анти-пример (осторожно с фейковыми цитатами).** Текст, который звучит как лендинг стартапа ("Your finance team is drowning in invoices...") — это маркетинг, а не голос пользователя. Помечай как сомнительное или выкидывай.

### Стартовая таксономия кластеров (из реального прогона по нашему домену)
Ориентир, не жёсткий список. Реальные кластеры с приоритетом из демо-прогона:
- **ИИ, который говорит, а не делает** (приоритет 0.95) — рядом: Excel, Microsoft Copilot, Office, OpenClaw, ChatGPT
- **Тону в почте** (приоритет 0.90) — рядом: Gmail, Outlook, Microsoft 365 Copilot, Claude, Slack
- **Автоматизация продаж и лидов** (приоритет 0.88) — рядом: CRM, Salesforce, HubSpot, Zendesk, Pipedrive
- **No-code сложен и дорог** (приоритет 0.82) — рядом: Zapier, Make.com, n8n, Integromat, Airtable
- **Надёжность и доверие к агенту** (приоритет 0.70) — рядом: ChatGPT, Claude, LLM, CRM, AI
- **Не хочу говорить с продавцом** (приоритет 0.55) — рядом: 

Эталонные углы (как формулировать), из того же прогона:
- _ИИ, который говорит, а не делает_ → 🎯 Хватит ИИ, который только рассказывает, как сделать работу. Наш агент сам подключается к вашим инструментам и выполняет задачи за вас — под вашим контролем.
- _Тону в почте_ → 🎯 Вы тонете в почте, а важные письма теряются между рассылками и счетами. Агент сам сортирует входящие, отвечает на типовые письма и переносит данные в нужные приложения.
- _Автоматизация продаж и лидов_ → 🎯 Ваши продавцы тратят часы на ввод данных в CRM и фоллоу-апы вместо продаж. Агент сам фиксирует звонки, заводит лиды, отправляет фоллоу-апы и отвечает за минуты, а не за часы.
