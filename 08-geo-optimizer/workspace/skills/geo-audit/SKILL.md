# GEO-аудит: цитируемость страницы

Цель: измерить, насколько страница пригодна для цитирования в ChatGPT/Perplexity/AI Overviews, и выдать приоритизированный план правок.

## Алгоритм
1. Открой страницу чтением веб-страниц. Если дан только домен — возьми главную + 2 ключевые страницы услуг (по запросам из BRAND_FACTS.md). Больше 5 страниц за один аудит не бери — предложи батчи.
2. Проверь техдоступность: открой `<домен>/robots.txt` — не заблокированы ли GPTBot, PerplexityBot, ClaudeBot, Google-Extended; открой `<домен>/llms.txt` — существует ли. Фиксируй дословно, что нашёл.
3. Пройди по всем пунктам knowledge/GEO_CHECKLIST.md. Каждому пункту — оценка: 0 (нет), 1 (частично), 2 (хорошо) + одна строка доказательства (цитата/факт со страницы).
4. Посчитай баллы по 5 измерениям и общий процент от максимума.
5. Citation-gap (если попросили или это тестовый прогон): по 1–2 запросам из BRAND_FACTS.md сделай веб-поиск, открой топ-3 страницы, отметь, какие пункты чек-листа у них закрыты, а у пользователя нет.
6. Составь план правок: топ-5 по правилу «влияние высокое, трудозатраты низкие — первыми». Для каждой правки: что сделать, где на странице, кто делает (копирайтер/разработчик), готовый фрагмент если применимо (llms.txt, строки robots.txt, JSON-LD).
7. Полный отчёт сохрани в Google Docs «GEO-аудит <домен> <дата>», строку с баллами добавь в Sheets «GEO-трекер видимости», лист «Аудиты».

## Выход (резюме в чат)
```
GEO-аудит: dentalux.ru/implantacia — 14/30 (47%)
Доступность 4/6 | Структура ответов 2/6 | Факты 3/6 | Разметка 2/6 | Авторитетность 3/6
Топ-правки:
1. Первый абзац не отвечает на запрос «сколько стоит имплантация» — переписать: прямой ответ с ценой в первых 2 фразах (копирайтер, 20 мин)
2. Нет FAQ-блока — добавить 5 вопросов из «Люди спрашивают» (готовы в отчёте)
3. robots.txt блокирует GPTBot (строка 12) — убрать Disallow (разработчик, 5 мин)
4. Нет llms.txt — файл готов в отчёте, положить в корень сайта
5. Цены «от ...» без конкретики — вставить вилку и срок из BRAND_FACTS.md
Полный отчёт: <ссылка на Docs>
```

## Краевые случаи
- Страница не открывается / закрыта от роботов → зафиксируй как критический вывод аудита (0 по доступности), попроси пользователя прислать текст страницы файлом или вставкой.
- Сайт на JS без серверного рендера (чтение возвращает пустоту) → отметь «контент недоступен краулерам — это и есть главная проблема», рекомендация: SSR/пререндер, аудит текста — по вставке от пользователя.
- Нет BRAND_FACTS.md или он пуст → аудит проводи, но в правках про факты ставь [НУЖНА ЦИФРА] и попроси заполнить файл.
- Повторный аудит той же страницы → сравни с прошлой строкой в листе «Аудиты», покажи дельту по измерениям.
