# GEO-чек-лист: аудит цитируемости (5 измерений, 15 пунктов)

Оценка каждого пункта: 0 — нет, 1 — частично, 2 — хорошо. Максимум 30. К каждой оценке — строка доказательства (цитата со страницы, строка robots.txt, скрин-факт).

## 1. Доступность для AI-краулеров (макс 6)
- 1.1. robots.txt не блокирует AI-ботов: GPTBot, PerplexityBot, ClaudeBot, Google-Extended, CCBot. 2 — все разрешены; 1 — часть; 0 — заблокированы или robots.txt отдаёт ошибку.
- 1.2. llms.txt в корне: краткое описание компании, ключевые страницы, факты. 2 — есть и актуален; 1 — есть, но пустой/устаревший; 0 — нет.
- 1.3. Контент доступен без JS: чтение страницы возвращает основной текст. 2 — весь текст в HTML; 1 — часть подгружается скриптами; 0 — без JS страница пустая.

## 2. Структура ответов (макс 6)
- 2.1. Первый абзац страницы прямо отвечает на главный запрос (ответ в первых 2–3 фразах, с конкретикой). 2 — да; 1 — ответ есть, но закопан ниже первого экрана; 0 — вводная вода.
- 2.2. H2/H3 сформулированы как вопросы пользователей, под каждым — самодостаточный абзац-ответ 40–70 слов. 2 — системно; 1 — местами; 0 — заголовки-этикетки («Наши преимущества»).
- 2.3. Списки и таблицы для перечислимого (цены, сравнения, этапы) — AI охотно цитирует структуры. 2 — есть там, где уместно; 1 — редко; 0 — сплошной текст.

## 3. Факты и определённость (макс 6)
- 3.1. Конкретные цифры вместо оценочных слов: цены/вилки, сроки, количество, проценты с указанием источника/года. 2 — ≥5 проверяемых фактов на странице; 1 — 1–4; 0 — только «большой опыт, лучшие цены».
- 3.2. Определённость формулировок: утверждения без хеджинга («ставим за 1 визит», а не «как правило, возможно, достаточно быстро»). 2 — да; 1 — смешанно; 0 — сплошные «возможно/обычно/индивидуально».
- 3.3. Сущность бренда однозначна: кто, где, для кого — единообразно на странице и совпадает с BRAND_FACTS.md. 2 — да; 1 — разнобой формулировок; 0 — непонятно, чем занимается компания.

## 4. Разметка (макс 6)
- 4.1. JSON-LD Organization/LocalBusiness с NAP (имя, адрес, телефон) и sameAs. 2 — есть и валиден; 1 — частичный; 0 — нет.
- 4.2. FAQ-блок на странице + разметка FAQPage. 2 — есть блок и разметка; 1 — блок без разметки; 0 — нет FAQ.
- 4.3. Мета и заголовки: title/H1 содержат запрос естественной фразой; страница про один интент. 2 — да; 1 — размыто; 0 — title-спам или нерелевантен.

## 5. Авторитетность (макс 6)
- 5.1. Автор/эксперт указан у контента (имя, регалии) или явное «кто мы» с проверяемыми фактами. 2 — да; 1 — обезличенно, но компания ясна; 0 — аноним.
- 5.2. Даты публикации/обновления видны, контент свежий (обновлялся за последние 12 мес). 2 — да; 1 — даты есть, контент старый; 0 — дат нет.
- 5.3. Внешние подтверждения: упоминания домена на сторонних площадках (каталоги, отзовики, СМИ) — проверить веб-поиском «<бренд> отзывы|упоминания». 2 — ≥3 живых площадки; 1 — 1–2; 0 — ноль.

## Интерпретация итога
- 24–30: страница цитируема, работать над охватом запросов.
- 15–23: середина — правки структуры и фактов дадут наибольший прирост.
- 0–14: сначала базовые блокеры (доступность, прямые ответы), остальное потом.

## Приоритизация правок (правило)
Сначала: блокеры доступности (1.1–1.3) — без них остальное не увидят. Затем: первый абзац (2.1) и факты (3.1) — максимальный эффект на цитируемость. Затем: FAQ+разметка (4.2). Последними: авторитетность — медленный контур.
