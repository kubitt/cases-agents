# Разбор договора и redline

> Скилл — одна повторяемая задача, описанная как рецепт. Шаги, лимиты и форматы можно править обычными словами.

Цель: за одну сессию — карта условий, риски против плейбука и готовые формулировки правок. Не пересказ, а решение.

## Алгоритм
1. Прочитай документ (PDF/DOCX). Зафиксируй: тип (NDA / услуги / поставка / SaaS / аренда / иное), стороны, наша роль (заказчик/исполнитель — сверь с профилем в LEGAL_PLAYBOOK.md). NDA → переключись на knowledge/NDA_FASTTRACK.md. Тип вне профиля (гос.закупки, M&A, международный спор) → стоп, эскалация с пометкой «вне профиля».
2. Извлеки условия по 10 блокам: предмет; цена и порядок оплаты; срок и автопролонгация; ответственность и лимиты; неустойки/штрафы; IP и результаты работ; конфиденциальность; расторжение и односторонний отказ; гарантии/SLA; подсудность и применимое право. Для каждого: номер пункта + цитата ≤25 слов. Блок отсутствует в договоре — тоже фиксируй: «нет лимита ответственности» = ответственность не ограничена → критично.
3. Сверь каждый блок с LEGAL_PLAYBOOK.md: ок / отклонение (есть запасная позиция) / критично (стоп-позиция) / нет позиции (плейбук молчит).
4. Для каждого отклонения подготовь redline-правку: цитата оригинала → готовый текст новой формулировки (не «нужно изменить», а вставляемый текст пункта) → одна фраза «почему» со ссылкой на позицию плейбука.
5. Создай в Google Docs копию «<имя файла> — REDLINE <дата>»: сверху таблица рисков, ниже текст с правками (разметка: ~~вычеркнуто~~ → **новая формулировка** + комментарий). Если исходник не переносится как текст (скан) — отдельный Doc только с таблицей правок по пунктам.
6. Добавь строку в Sheets «Реестр договоров»: контрагент, тип, срок, auto_renewal, renewal_notice_days, notice_deadline, статус review_done, ссылки.
7. Передай результат скиллу risk-verdict.

## Формат таблицы рисков
- 7.2 — «Исполнитель возмещает все убытки, включая упущенную выгоду» — неограниченные выплаты при любом сбое — критично — «Совокупная ответственность Исполнителя ограничена стоимостью договора; упущенная выгода возмещению не подлежит»
- 4.1 — «оплата в течение 90 дней» — кассовый разрыв — отклонение — «в течение 30 календарных дней» (запасная: 45)

## Краевые случаи
- Скан без текстового слоя → попроси DOCX или текстовый PDF; по картинке молча не разбирай.
- Договор на английском → разбор и memo на русском, формулировки правок — на языке договора.
- Файл >50 страниц → сначала извлеки 10 блоков по оглавлению, спроси пользователя о приоритетных разделах, затем сплошная проверка блоков ответственности/оплаты/расторжения.
- Ссылки на приложения, которых нет в пакете → флаг «нет приложения N — запросить у контрагента», в вердикте учесть как открытый риск.
