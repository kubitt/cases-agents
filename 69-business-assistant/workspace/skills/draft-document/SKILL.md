# Draft a business document

> A skill = one repeatable job, written as a recipe. You can edit steps, limits, and formats in plain words.

Goal: a ready-to-review draft that sounds like the user, not like a marketer — clear, specific, and honest, with sources for any claim.

## Algorithm
1. Read knowledge/BUSINESS_PROFILE.md for the offer, audience, and brand voice. Match the voice; never override it with a generic tone.
2. Confirm four things (ask only what's missing): what it's for, the audience (cold or warm), the one main message or offer, and where it will run (email, Instagram, ad, one-pager).
3. Pick the frame by funnel stage: cold/awareness (TOFU) — hook the problem; considering (MOFU) — proof and comparison; ready-to-buy (BOFU) — offer and clear call to action. Use a simple copy framework — AIDA, PAS, or BAB — and name which one.
4. Write the draft. For ads, make the first line stand alone and under ~125 characters (the part shown before "See more"). One clear call to action per piece. Every factual claim gets a source URL; no claim you cannot source.
5. Save it to a Google Doc. For an email, also create an unsent Gmail draft and ask for approval. If it's planned content, add a row to Google Sheets "Content Plan" (columns: publish_date, channel, title, status, doc_link, notes) and a Google Calendar event on publish_date.

## Output
```
Type: Instagram caption (warm audience, framework: PAS)
Hook (118 chars): Beans going stale before you finish the bag? That's flavor — and money — down the drain.
Body: Our subscription ships small-batch beans the week they're roasted, so every cup tastes like the first...
CTA: Tap to start a subscription — pause anytime.
Claims sourced: roast-freshness window (source: ...).
Saved: Google Doc "Subscription caption — Jul"; Content Plan row added (publish_date 2026-07-10). Approve to keep?
```

## Edge cases
- Voice file empty → write one plain, friendly draft, mark it "voice not set", and ask for one real sample to learn from.
- A requested claim can't be sourced → drop it or rewrite around it; never state an unverifiable number as fact.
- Request looks like a fake review, fake testimonial, or fake urgency → refuse and explain the reputational and legal risk, then offer an honest alternative.
- Email draft ready → it stays unsent in Gmail until the user approves; sending is never automatic.
