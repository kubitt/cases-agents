# Архитектор кампаний

> Скилл — одна повторяемая задача, описанная как рецепт. Шаги, лимиты и форматы можно править обычными словами.

Цель: карта сегментов и веток, где у каждого письма есть работа. Ветки не пересекаются в один день у одного контакта.

## Алгоритм
1. Из описания бизнеса (USER.md) определи жизненный цикл клиента: как узнают → первая покупка → повторная → уход. Для каждого перехода — где теряются люди.
2. Спроектируй ветки под потери. Базовый набор: welcome (подписался, не купил; 3–5 писем), activation (купил, не воспользовался; 2–3), winback (нет покупок N дней; 2–3), VIP (топ-10% по чеку; 1–2 в квартал). Не плоди больше 4–5 веток на старте.
3. Для каждой ветки: условие входа (событие/сегмент), условие выхода (цель достигнута / стоп-слово / отписка), количество писем, задержки (дефолт: 0ч / 2д / 5д / 9д), цель в цифре (например «10% дошли до покупки»).
4. Правило приоритета: контакт подходит под две ветки → активна одна, порядок приоритета welcome > activation > winback > VIP.
5. Запиши в Sheets «Lifecycle-кампании»: лист map (ветка, цель, условия, письма, статус) и лист settings (строка = письмо: branch, step, delay, subject_ref, goal, stop_conditions).

## Выход
Карта в Sheets + сообщение: схема веток в 10 строках, что писать первым и почему.

## Краевые случаи
- B2B с длинным циклом → задержки ×2, меньше писем, больше пользы в каждом.
- Нет событийных данных (неоткуда узнать «купил») → упрощай до сегментов по дате подписки/покупки из выгрузки базы; отметь ограничение в map.
