# Бенчмарк зарплаты

> Скилл — одна повторяемая задача, описанная как рецепт. Шаги, лимиты и форматы можно править обычными словами.

Цель: защищаемая вилка p25/p50/p75 — с источниками, датой и честными оговорками. Не «средняя по больнице».

## Алгоритм
1. Нормализуй роль: синонимы названий (маркетолог = digital-маркетолог = growth …) — ищи по 2–3 вариантам. Грейд уточни по 90-дневным результатам из профиля роли (junior/middle/senior).
2. Источники (веб-поиск, свежесть ≤6 месяцев): агрегаторы вакансий (заявленные вилки по региону), обзоры зарплат (публичные исследования), телеграм-каналы вакансий ниши. Минимум 8–10 точек данных; каждую фиксируй: сумма/вилка, источник, URL, дата.
3. Очисти выбросы (очевидно другой грейд/регион). Посчитай p25/p50/p75 по точкам. При <8 точках — пометь «мало данных, ориентировочно».
4. Поправки — явно и отдельно: удалёнка vs офис, белая/серая (если рынок такой — скажи прямо), опционы/бонусы за скобками.
5. Выход: таблица точек данных (в Sheets, лист роли) + резюме: «p25/p50/p75 = X/Y/Z <валюта> на руки, N точек, источники: …. Ваше ощущение (из профиля роли) — попадает в pXX». Рекомендация: целиться в p50–p60 при обычной срочности, p75 — при горящей.
6. Обнови строку в «Реестре найма»; при повторном бенчмарке покажи дельту к прошлому.

## Краевые случаи
- Разброс источников >2x → покажи бимодальность («две разные роли под одним названием»), предложи уточнить грейд.
- Регион без данных → ближайший рынок с поправочным коэффициентом, коэффициент — с обоснованием.
- Пользователь просит «напиши вилку пониже, чтоб сэкономить» → вилка рыночная в ките, решение об оффере — его; последствия (время найма, отказы) — одной строкой.
