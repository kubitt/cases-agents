# 13-недельный прогноз кэша

Цель: одна страница — сколько денег, куда идут, когда закончатся при трёх сценариях.

## Алгоритм
1. Week-0 = фактический остаток на счетах из последней выписки/Sheets «Финансы». Дата остатка обязательна в отчёте. Остаток старше 7 дней — предупреди.
2. Поступления по неделям: подтверждённые счета (реестр дебиторки, если есть) по срокам оплаты со скидкой на надёжность (дефолт: 90% в срок, хвост +2 недели), повторяющаяся выручка — по истории 3 месяцев.
3. Списания по неделям: регулярные (зарплаты, аренда, подписки — из CHART_OF_ACCOUNTS.md, раздел «Регулярные платежи») + разовые известные (пользователь сообщает) + средний прочий burn.
4. Три сценария: base (как есть), stress (поступления −30%, задержка +2 недели), upside (+20% или крупная сделка пользователя). Runway в неделях для каждого.
5. Выход — Sheets «Runway» (13 колонок-недель × строки статей, три блока сценариев) + сообщение: остаток, runway base/stress, ближайшая «яма» (неделя с минимальным остатком), 1–2 действия (например: «сдвинуть платёж X, ускорить счёт Y — яма недели 6 исчезает»).
6. Каждую строку прогноза привязывай к источнику (счёт №, категория, история). Строк «прочее» больше 20% от burn быть не должно — иначе пометь «уточнить категоризацию».

## Краевые случаи
- Runway base < 8 недель → первая строка сообщения: «Внимание: runway N недель», без смягчений.
- Нет реестра дебиторки → поступления только по истории, пометь снижение точности и предложи вести реестр.
- Разовые крупные суммы в истории → исключи из среднего burn, покажи отдельно.
