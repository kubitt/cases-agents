# Trend research

> A skill = one repeatable job, written as a recipe. You can edit steps, limits, and formats in plain words.

Goal: produce 5–8 concrete post topics for the week, each grounded in a real trend or fact — so post-writer never invents.

## Algorithm
1. Read the target brand profile from knowledge/BRAND_PROFILE.md (niche, audience, banned topics). If several profiles exist, use the one named in the request.
2. Search the last 7 days: web search (or Firecrawl if connected) for "<niche> news", "<niche> trends", "<niche> tips". Read 3–5 real sources.
3. Pull candidate angles: fresh news, a recurring question in the niche, a seasonal moment, a myth to bust, a quick how-to.
4. Cross-check the content log (knowledge/CONTENT_PLAYBOOK.md history + "SMM Content Log" sheet): drop anything covered in the last 4 weeks.
5. Rank 5–8 topics by relevance to the audience's pains and by freshness. For each, note the source and the content format (tip / list / story / question / case).
6. Hand the topic list to post-writer, or show it if the user only asked for topics.

## Output (topic list handed to post-writer)
```
Week topics — niche: home coffee brewing (source-backed)
1. [how-to] New pour-over ratio going around this week — src: url
2. [myth] "Dark roast = more caffeine" — bust it — src: url
3. [list] 3 cheap grinders under $50 people recommend now — src: url
4. [question] Aeropress vs V60 — start a debate
5. [seasonal] Iced coffee recipes for the heatwave — src: url
```

## Edge cases
- Thin search results → rely on evergreen niche knowledge, mark topics "evergreen (no fresh trend)", still avoid recent repeats.
- Banned topic surfaces as a trend → skip it, note why; never pitch a topic the brand forbids.
- Multiple brand profiles → research per the requested profile only; never blend niches.
- Fast-moving niche (news/crypto) → prefer items <48h old and label the date, since they age quickly.
