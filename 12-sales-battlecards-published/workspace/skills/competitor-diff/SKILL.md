# Дифф конкурента

> Скилл — одна повторяемая задача, описанная как рецепт. Шаги, лимиты и форматы можно править обычными словами.

Цель: найти значимые изменения у конкурента с прошлой проверки и превратить их в конкретные правки карточек — а не в «новостную сводку».

## Алгоритм
1. Из Sheets «Реестр конкурентов», лист snapshots, возьми последний снимок конкурента: дата, оффер (заголовок сайта), тарифы и цены, список продуктов/фич, ключевые новости.
2. Собери свежий снимок тем же методом: сайт (главная, тарифы, продукты) + веб-поиск новостей за период с прошлого снимка.
3. Сравни по секциям: цены и тарифы / продукты и фичи / оффер и мессаджинг / новости (раунд, релиз, ключевые наймы, крупные клиенты) / вакансии.
4. Классифицируй каждое изменение: critical (цены, новый продукт, раунд, выход в наш сегмент) / notable (новый мессаджинг, крупный кейс, заметный найм) / minor (косметика, мелкие фичи). Minor в дайджест не включай, если их меньше трёх.
5. Для каждого critical/notable укажи: какую секцию какой карточки обновить + черновик новой формулировки (1–2 фразы).
6. Дайджест пользователю на утверждение. После «ок»: правки в Google Docs карточек, строка в changelog дока («2026-07-03: тариф Pro 490→590, обновлён скрипт "торгуется по цене"»), новый снимок в snapshots (старые не удаляй — это история), last_snapshot в реестре.

## Выход (дайджест)
```
Мониторинг конкурентов, 04.07
1. TaskFlow — critical — тариф Pro: 490→590 руб/место (taskflow.ru/pricing, вчера). Обновить: «Цены» + скрипт №3. Черновик: «разница в цене сократилась до 15% — аргумент "у них дешевле" ослаб».
2. TaskFlow — notable — кейс с X5 на главной. Обновить: «Позиционирование».
3. Планфикс — без изменений.
Обновить карточки: все / номера?
```

## Краевые случаи
- Сайт конкурента недоступен → пометь «сайт недоступен, проверю в следующий прогон»; изменением не считай, карточку не трогай.
- Первый прогон (прошлого снимка нет) → создай базовый снимок, в дайджест: «снимок создан, дифф — со следующей недели».
- Изменений слишком много (редизайн сайта) → не перечисляй всё, предложи: «похоже на редизайн/репозиционирование — пересобрать карточку целиком (battlecard-builder)?».
- Пользователь не ответил на дайджест → карточки не правь; неутверждённые изменения накапливай и покажи в следующем дайджесте отдельным блоком «с прошлой недели без решения».
