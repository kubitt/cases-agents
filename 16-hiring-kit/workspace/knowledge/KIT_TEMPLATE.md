# Структура кита (папка «Кит: <роль>» в Google Docs)

> Ваш справочный файл. Агент читает его перед работой и дополняет по ходу. Правьте свободно.

## Документ 1. JD
О компании (5 строк, из профиля) → Миссия роли (1 фраза: зачем существует) → Результаты 90 дней (3 пункта, проверяемые) → Задачи (5–8) → Must-have (≤5, каждый проверяем на интервью) → Nice-to-have → Условия (вилка после бенчмарка, формат, бонусы) → Как откликнуться (что прислать, что будет дальше, сроки).

## Документ 2. Вилка (бенчмарк)
Резюме p25/p50/p75 → таблица точек данных с источниками и датами → поправки → рекомендация по позиционированию оффера.

## Документ 3. Воронка и вопросы
Схема 4 этапов (цель, длительность, кто ведёт, порог) → банки вопросов по этапам: вопрос + «что слушать» → правила кейса/тестового.

## Документ 4. Scorecards
Таблица на этап: критерий (из JD) | вопросы-источники | шкала 1–4 с описаниями | вес. Правило решения по итогам. Правило калибровки: первые 3 кандидата — оценка двумя интервьюерами независимо.

## Документ 5. Постинги
hh-версия / telegram-версия (≤800 знаков) / LinkedIn-стиль (если нужен) + чек-лист публикации (куда, когда, бюджет продвижения — если пользователь спросит).

## Правило согласованности
Любая правка must-have в JD тянет обновление вопросов и scorecards — агент делает это автоматически и сообщает.
