# Автоматизации: Хайринг-кит

Агент в основном событийный (one-shot по команде). Постоянных cron нет.

## Триггер (опционально, создаётся при онбординге по желанию)

### 1. Новая вакансия в Drive
- Событие: Google Drive — новый файл в папке «Вакансии», имя начинается с «вакансия-» (например «вакансия-маркетолог.docx» с черновыми заметками о роли).
- Действие: «Прочитай файл. Извлеки роль, пожелания, особенности. Собери полный кит по скиллу role-kit (+ бенчмарк по скиллу comp-benchmark). Уточняющие вопросы (максимум 3) задай пользователю ДО генерации, если в файле нет ответа про задачи первых 90 дней. Готовый кит — папкой в Drive "Вакансии/<роль>", ссылки и вилку — пользователю.»
- Кредиты: срабатывает только на файлы по маске — расход низкий.

## Разовая задача при обновлении бенчмарка
По команде «обнови вилку по <роль>» — пересчитать бенчмарк, показать дельту к прошлому (из реестра в Sheets). Постоянный cron не нужен: зарплатные данные достаточно обновлять под конкретный найм.

Тихие часы 23:00–08:00 соблюдаются.
