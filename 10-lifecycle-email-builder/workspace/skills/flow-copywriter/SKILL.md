# Копирайтер веток

> Скилл — одна повторяемая задача, описанная как рецепт. Шаги, лимиты и форматы можно править обычными словами.

Цель: письма, которые открывают и дочитывают. Голос бренда, один CTA, ноль спам-приёмов.

## Алгоритм
1. Возьми ветку из map/settings и тон из knowledge/BRAND_VOICE.md.
2. На каждое письмо: тема ≤45 знаков + 2 A/B-варианта (разные механики: вопрос / выгода / интрига без обмана); прехедер ≤80 знаков; тело 80–200 слов; один CTA (кнопка/ссылка, глагол действия); PS по необходимости.
3. Драматургия ветки: письмо 1 — мгновенная ценность (обещанное за подписку/покупку), середина — польза + социальное доказательство, финал — прямой оффер с честным дедлайном (если дедлайн реальный).
4. Персонализация: только поля, реально существующие в базе (проверь колонки листа queue/выгрузки). Плейсхолдер формата {{first_name}} с fallback-вариантом без имени.
5. Каждое письмо заканчивается блоком отписки (текст под тон бренда, но явный).
6. Сохрани в Google Docs «Ветка: <название>»: письма по порядку, у каждого — тема/A/B/прехедер/тело; в settings проставь subject_ref.

## Выход
Док с текстами + сообщение: ветка N писем готова, что утвердить.

## Краевые случаи
- Пользователь просит агрессивную срочность без реального дедлайна → предложи честную альтернативу (ограничение мест/бонуса), при настаивании — правило 2 SOUL.
- Правка одного письма ломает драматургию → предупреди и предложи скорректировать соседние.
