# Сборка ответа на RFP

> Скилл — одна повторяемая задача, описанная как рецепт. Шаги, лимиты и форматы можно править обычными словами.

Цель: полный ответ по структуре заказчика, где каждое обязательное требование адресовано явно, а текст звучит словами заказчика, не «AI-прозой».

## Алгоритм
1. Возьми чек-лист из rfp-parse. Структура ответа = требуемая структура заказчика; если не задана — дефолт: резюме (1 стр.) → о компании → понимание задачи → решение → план и сроки → команда → опыт и кейсы → цена [ДАННЫЕ ОТ ВАС] → приложения.
2. Определи 3 win-темы: пересечение «что важно заказчику» (критерии оценки, боли из текста RFP) × «наши доказуемые сильные стороны» (кейсы библиотеки). Пример: «уже внедряли в этой отрасли», «локальная команда — реакция в день обращения», «фикс-цена без допсоглашений». Зафиксируй их комментарием в начале черновика; каждая секция должна поддерживать хотя бы одну.
3. Собирай секцию за секцией: подбери материал (COMPANY_LIBRARY.md + релевантные фрагменты прошлых КП из Drive «Тендеры/Библиотека КП» — ищи по предмету и отрасли) → перепиши терминологией заказчика: используй ИХ слова (в RFP «подрядчик» — пишем «подрядчик», не «партнёр»; их названия этапов, ролей, систем).
4. На каждое обязательное требование — явный ответ-зеркало: «Требование 4.1.2 (опыт от 3 лет): подтверждаем — 6 лет, кейсы №2, №5 в разделе "Опыт"». Обновляй колонку «Секция ответа» в чек-листе.
5. Всё, чего нет в библиотеке, — пометка [ДАННЫЕ ОТ ВАС: что именно]. Собери их единым списком «Дозаполнить» в конце документа.
6. Оформи в Docs «<Тендер> — Ответ v1» в папке «Тендеры/Готовые ответы». Объём — по лимиту RFP; лимита нет — 8–15 страниц, резюме ≤1 страницы.
7. Прогони knowledge/COMPLIANCE_CHECKLIST.md, приложи результат к сводке.

## Правила языка
- Активный залог, цифры из кейсов с контекстом («сократили срок согласования с 14 до 3 дней»).
- Без превосходных степеней без доказательств: «лидер рынка» — только с фактом и источником.
- Запрещено: копировать текст чужих КП из веба, обещать услуги вне библиотеки, оставлять плейсхолдеры прошлых заказчиков.

## Выход (сводка пользователю)
```
Ответ собран: <ссылка на Doc>
Секций: 9/9. Обязательных требований адресовано: 14/14.
[ДАННЫЕ ОТ ВАС]: 3 позиции — цена (раздел 8), ФИО руководителя проекта, скан ISO.
Компл-чек: OK 12 | ПРОВАЛ 1 (формат PDF — экспортировать) | ВРУЧНУЮ 2.
Следующий шаг: заполните 3 позиции — финализирую и прогоню компл-чек повторно.
```

## Краевые случаи
- Библиотека пустая/бедная → собери каркас с пометками и скажи прямо: «ответ слабый без ваших кейсов — добавьте 2–3».
- Жёсткий лимит объёма → сокращай кейсы и «о компании», НЕ сокращай ответы на обязательные требования.
- Заказчик требует свои формы (таблицы Word) → тексты для форм отдельным блоком «перенести в форму №N» — заполняет пользователь.
- Повторный тендер того же заказчика → подними прошлый ответ и результат из реестра, учти причину проигрыша, если известна.
