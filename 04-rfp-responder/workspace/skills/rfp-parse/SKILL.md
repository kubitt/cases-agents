# Разбор RFP в чек-лист соответствия

> Скилл — одна повторяемая задача, описанная как рецепт. Шаги, лимиты и форматы можно править обычными словами.

Цель: за один проход — все обязательные требования, дедлайны и честный go/no-go. Ни одного «сюрприза» на этапе подачи.

## Алгоритм
1. Прочитай ВСЕ файлы пакета (RFP, ТЗ, проект договора, формы, разъяснения). Зафиксируй: заказчик, предмет, бюджет/НМЦ, дедлайн подачи (дата, время, часовой пояс), способ подачи (площадка/email/бумага), дедлайн вопросов заказчику.
2. Извлеки обязательные требования — маркеры: «должен», «обязан», «не допускается», «обязательно», «требуется», shall/must. Отдельно квалификационные: опыт N лет, аналогичные проекты, штат, лицензии/сертификаты, обороты, обеспечение заявки/контракта. Каждое — строкой с номером пункта источника.
3. Извлеки критерии оценки и веса (цена X%, качество Y%, срок Z%), требуемую структуру ответа (разделы, обязательные формы), ограничения формата (объём, язык, формат файла, число копий).
4. Построй чек-лист соответствия (формат ниже). Соответствие оценивай по COMPANY_LIBRARY.md (раздел «Квалификация») и файлам библиотеки. Не уверен — ставь «вопрос», не «да».
5. Сводка go/no-go: смертельные несоответствия (обязательное + «нет»), закрываемые («частично»), вопросы заказчику. Рекомендация: GO / NO-GO / GO при условии. Решение — за пользователем.
6. Сохрани чек-лист в Docs «<Тендер> — чек-лист», строку в реестр (Sheets, если подключён), предложи разовое напоминание за 2 дня до дедлайна.

## Формат чек-листа
- 1 — опыт аналогичных проектов от 3 лет — 4.1.2 — да — да — кейсы №2, №5 — Опыт
- 2 — наличие сертификата ISO 9001 — 4.1.5 — да — нет — — — СТОП-фактор
- 3 — план-график работ понедельно — 6.3 — да — да — шаблон из КП-2025-03 — План

## Формат сводки
```
ТЕНДЕР: Заказчик X — внедрение СЭД — НМЦ 4,2 млн ₽
Дедлайн: 18.07 15:00 МСК (осталось 9 раб. дней). Вопросы заказчику — до 11.07.
Обязательных требований: 14. Да: 11, частично: 2, нет: 1 (ISO 9001).
Критерии: цена 60%, опыт 25%, срок 15%.
Рекомендация: GO при условии — уточнить у заказчика, допускается ли сертификат в процессе получения (черновик вопроса готов).
```

## Краевые случаи
- Скан без текстового слоя → запроси читаемую версию; по картинке молча не разбирай.
- Противоречия в пакете (в ТЗ одно, в проекте договора другое) → отдельный блок «Противоречия» + в вопросы заказчику.
- Дедлайн < 5 рабочих дней → первая строка сводки: «СРОКИ КРИТИЧНЫЕ», объём работ по сборке оцени сразу.
- Пакет > 100 страниц → сначала целевое извлечение по оглавлению, затем сплошной поиск маркеров обязательности; отметь, какие файлы прочитаны.
- RFP на английском → чек-лист на русском, цитаты на языке оригинала, ответ готовится на языке RFP.
