# RFP/тендер-ответчик / RFP Responder

RFP или тендерная документация → чек-лист требований с go/no-go и собранный секция-за-секцией ответ на языке заказчика — за одну сессию вместо 8–20 часов.

## Ценность и ROI

Подготовка одного ответа на RFP — 8–20 часов работы ($400–2000). Агент сжимает это до одной сессии: разбор требований, каркас ответа из вашей библиотеки КП, проверка на пропущенные обязательные пункты. Один выигранный тендер окупает годы подписки $29/мес. Окупаемость 15–70x; плюс перестаёте пропускать дедлайны и «слетать» с тендеров по формальным причинам.

## Что делает

1. Читает RFP/ТЗ/тендерный пакет (PDF/DOCX из Drive-папки или файлом в чат) и разбирает в чек-лист: обязательные требования, квалификационные условия, дедлайны, формат подачи, критерии оценки с весами.
2. Даёт go/no-go сводку: проходим ли по обязательным условиям, что смертельно, что закрываемо, какие вопросы задать заказчику.
3. Собирает ответ секция-за-секцией в Google Docs из вашей библиотеки (knowledge/COMPANY_LIBRARY.md + прошлые КП в Drive) — терминологией заказчика.
4. Прошивает 3 win-темы через весь текст: почему вы, с доказательствами из кейсов.
5. Всё, чего нет в библиотеке, помечает [ДАННЫЕ ОТ ВАС] — ничего не выдумывает.
6. Перед сдачей прогоняет компл-чек (knowledge/COMPLIANCE_CHECKLIST.md): каждый обязательный пункт адресован, формат, приложения, дедлайн.
7. Опционально по понедельникам ищет новые тендеры по вашей нише через веб-поиск.

## Состав папки

- `workspace/` — копируется в воркспейс агента ASCN (IDENTITY, SOUL, ONBOARDING, скиллы, библиотека, компл-чек).
- `DEPLOY_PROMPT.md` — промпт для Claude Code, который переносит всё в ASCN.
- `INTEGRATIONS.md`, `AUTOMATIONS.md` — справка по интеграциям и текст опциональной cron-задачи.

## Быстрый деплой

1. Откройте Claude Code в этой папке.
2. Вставьте содержимое `DEPLOY_PROMPT.md`.
3. Напишите агенту «Привет, начнём настройку» — дальше он сам проведёт онбординг (интеграции, библиотека компании, разбор первого RFP).

## Требуемые интеграции

Google Drive (обязательно), Google Docs (обязательно). Google Sheets, Gmail — опционально. Канал: Telegram (рекомендуется). Веб-поиск — встроенный.

## Ограничения (сознательные)

- Не подаёт заявки на площадках (нет браузерной автоматизации) и не работает с ЭЦП — подача за вами; агент готовит пакет и напоминает о дедлайне.
- Без API тендерных агрегаторов — мониторинг через открытый веб-поиск: находит публичные объявления, полноту не гарантирует.
- Цены не назначает: ценовой раздел заполняете вы, агент только оформляет.
- Кейсы, цифры и сертификаты — только из вашей библиотеки: пробелы помечает, а не сочиняет.
