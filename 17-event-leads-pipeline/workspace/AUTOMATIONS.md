# Автоматизации: Бейджи с выставки → пайплайн

> Файл живёт в воркспейсе агента. Агент читает его при создании и изменении cron-задач. Расписания и формулировки можно менять — агент будет следовать обновлённой версии.

## Триггеры

### 1. Новый файл сканов (Google Drive)
- Событие: новый CSV/XLSX в папке «Выставки».
- Действие: «Обработай файл по скиллу badge-enrich: нормализация, дедуп, обогащение, скоринг A/B/C, запиши в Sheets "Ивент: <имя файла>". Пришли пользователю сводку (всего / A / B / C, топ-10 горячих с одной строкой почему) и спроси подтверждение на генерацию follow-up для сегмента A сегодня же (скилл followup-sequencer).»
- Кредиты: срабатывает только на файлы в одной папке — расход низкий.

### 2. Ответ контакта (Gmail: новое письмо) — включается на период кампании, выключается после
- Условие: отправитель есть в активном листе «Ивент: *» со статусом contacted.
- Действие: «Классифицируй (интерес / позже / отказ), обнови статус, при интересе — черновик ответа + алерт в Telegram немедленно.»

## Динамические задачи (создаёт сам агент после утверждения кампании, на 5–12 дней)

### 3. Ежедневный прогон цепочек
- Cron: `0 10 * * *` (10:00, живёт до конца кампании — агент удаляет задачу сам по завершении)
- Промпт: «По активным листам "Ивент: *": контакты, у которых сегодня срок следующего касания, — подготовь черновики по followup-sequencer, сводку на утверждение в Telegram. Ответившим касания останови. В конце кампании (все цепочки завершены) — финальный отчёт: контакты → ответы → встречи, конверсия по сегментам; предложи удалить эту задачу и триггер №2.»

Тихие часы 23:00–08:00 соблюдаются.
