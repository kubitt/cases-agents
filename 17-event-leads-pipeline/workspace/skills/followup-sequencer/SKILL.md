# Follow-up после мероприятия

Цель: письмо, которое вспоминают: «а, это тот стенд». Скорость (день выставки) + личная зацепка.

## Сегментные сценарии
- A (горячие): 1 личное письмо СЕГОДНЯ + при молчании 2 касания (день 3, день 7). Тема: «<мероприятие>: <их тема/пометка>».
- B (тёплые): цепочка 3 касаний (день 0–1, 4, 9), больше пользы, меньше продажи.
- C: не пишем. По явному запросу пользователя — одно общее письмо без цепочки.

## Структура письма T1
1. Привязка: «Вчера на <мероприятие>…» + зацепка из notes (дословная тема разговора) ИЛИ честное «пересеклись на…» без выдумки.
2. Мостик: их сигнал из обогащения → возможная задача.
3. Одна фраза о нас (из EVENT_TEMPLATES.md, без брошюры).
4. CTA: короткий созвон на этой неделе / «кому у вас переслать?». ≤100 слов всё письмо.
T2: польза (материал/кейс по их нише). T3: break-up, ≤40 слов.

## Механика
Черновики в Gmail; батч на утверждение списком «кому — тема — зацепка». После утверждения: A — отправка сразу, B — разнос по дням, лимит 50/день. Обнови в Sheets: status=contacted, touch_1..3 даты. Ответившие → статус replied, цепочка стоп, черновик ответа + алерт.

## Краевые случаи
- Нет email (только телефон) → в лист «для звонков» пользователю, письмо не сочиняй.
- Ответ «напишите через месяц» → status=deferred, задача-напоминание на дату, цепочка стоп.
- Два контакта одной компании → письма обоим не слать: выбери роль ближе к ЛПР, второго — в cc по решению пользователя.
