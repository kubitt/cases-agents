# Обогащение сканов

Цель: из сырого CSV — пайплайн, где про каждого понятно: кто, чем живёт компания, насколько горяч.

## Алгоритм
1. Маппинг колонок: найди в файле имя, компанию, должность, email, телефон, пометки (notes/comment — часто там контекст разговора у стенда). Покажи маппинг пользователю одной таблицей — подтверждение перед обработкой.
2. Нормализация: регистр имён, юрформы из названий компаний (ООО/LLC — вынести), домены email. Дедуп: одинаковый email = один контакт; одинаковая компания без email — пометь для ручного разбора.
3. Обогащение по компании (веб, ≤5 минут на контакт): что делает, размер (сотрудники/филиалы, если видно), свежая новость/сигнал. Источник каждого факта — URL в колонку sources.
4. Скоринг по критериям из knowledge/ENRICH_SCHEMA.md: A (совпадение ниши+размера+роль ЛПР), B (частичное), C (не ICP). К оценке — одна строка обоснования.
5. Запиши в Sheets «Ивент: <название>» по схеме. Пометки со стенда (notes) сохрани дословно — это золото для писем.
6. Сводка пользователю: всего / дублей убрано / A / B / C, топ-10 A с обоснованием, сколько без email (телефон есть — пометь для ручного звонка пользователем).

## Приоритет скорости
Файл >100 строк: сначала обработай и выдай сегмент A полностью (обогащение+скоринг), потом добей B и C фоном. Горячие письма должны уйти сегодня.

## Краевые случаи
- Кривая кодировка/разделители CSV → определи автоматически; не вышло — попроси пересохранить в UTF-8, объясни как.
- Фото визиток текстом в чат → распознай в те же колонки, пометь confidence.
- Компания не находится в вебе → скоринг только по должности и пометкам, confidence=low, в ручной разбор.
