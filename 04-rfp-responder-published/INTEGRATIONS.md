# Интеграции: RFP/тендер-ответчик

| Интеграция | Обязательность | Зачем | Права |
|---|---|---|---|
| Google Drive | Обязательно | Папки «Тендеры/Входящие», «Тендеры/Библиотека КП», «Тендеры/Готовые ответы» | Чтение и запись |
| Google Docs | Обязательно | Сборка ответа, чек-листы соответствия, вопросы заказчику | Чтение и запись |
| Google Sheets | Опционально | Реестр тендеров: дедлайны, статусы go/no-go, результаты | Чтение и запись |
| Gmail | Опционально | Черновики: запрос разъяснений заказчику, письмо подачи | Черновики |
| Telegram (канал) | Рекомендуется | Понедельничная подборка тендеров, напоминания о дедлайнах | Канал доставки |

Встроенные веб-поиск и чтение страниц — для мониторинга тендеров и ресёрча заказчика; OAuth не нужен.

Порядок подключения при онбординге: Drive → Docs → (Sheets, Gmail по желанию) → Telegram.

Без Sheets: нет реестра — статусы тендеров агент держит в памяти текущей работы, история не копится. Без Gmail: тексты писем в чат, отправляете сами. Без Telegram: всё в чат ASCN.

Триггеров нет сознательно: RFP приходят нечасто, а триггер Drive жёг бы кредиты на каждом файле библиотеки. Запуск — файлом в чат, командой «разбери <файл в Drive>» или через понедельничный мониторинг. Чего нет: API тендерных площадок и ЭЦП — подача заявки всегда за пользователем; расширение — кастомный MCP к агрегатору тендеров.
