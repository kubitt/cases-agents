# KPI-отчёт

> Скилл — одна повторяемая задача, описанная как рецепт. Шаги, лимиты и форматы можно править обычными словами.

Цель: отчёт, который руководитель читает за 3 минуты и знает, что делать. Не дамп цифр.

## Алгоритм
1. Прочитай knowledge/METRICS_MAP.md: список метрик, источники, периоды, цели, допуски, кластеры, направления.
2. Собери значения за отчётный период: Sheets — указанные листы/колонки по дате периода; GSC — клики, показы, CTR, средняя позиция за период и прошлый период; GitHub — merged PR, релизы, открытые issues за период.
3. Допиши значения в таблицу «KPI-история», лист history (date, metric, value), если за этот период их там ещё нет — история нужна для трендов и дашборда.
4. Посчитай по каждой метрике: дельту к прошлому периоду (WoW для недельного, MoM для месячного) в процентах и дельту к цели, если цель задана.
5. Отклонение больше допуска (дефолт 20%) → комментарий 1–2 фразы. Сначала посмотри разбивку в источнике (по каналам/продуктам/дням, если колонки есть), назови вероятную причину; причины из данных не видно — честно: «причина не видна из данных, проверьте X».
6. На каждый кластер — ровно ОДНО действие: самое важное, конкретное, с адресатом («маркетингу: вернуть старую форму заявки до пятницы»). Кластер в норме → пиши «в норме, действий не требуется», действие «на всякий случай» не выдумывай.
7. Собери отчёт по knowledge/REPORT_STRUCTURE.md (недельный или месячный раздел). Отправь в канал. Затем вызови скилл html-dashboard и приложи ссылку на обновлённый файл.

## Формат KPI-таблицы
```
- Выручка, тыс руб — 1 240 — 1 400 — +4% — ниже плана −11%
- Новые лиды — 52 — 80 — −34% — отклонение, см. п.1
- Клики из поиска — 8 310 — — — +6% — в норме
```
Статусы: в норме / ниже плана N% / отклонение WoW N% / n/d.

## Краевые случаи
- Источник недоступен или лист переименован → метрика «n/d: нет доступа к <адрес>», отчёт не блокируй, в конце блок «Проблемы с данными» с инструкцией, что починить.
- Метрика без цели → сравнивай только с прошлым периодом.
- Аномалия (×3+ к медиане истории) → перепроверь источник повторным чтением, пометь «возможна ошибка данных: проверьте <ячейка>», в действия не бери.
- Истории меньше 2 периодов → без дельт, только факт + пометка «база накапливается».
- Все метрики n/d (протух OAuth) → отчёт не отправляй, отправь алерт «источники недоступны, переподключите Sheets».
