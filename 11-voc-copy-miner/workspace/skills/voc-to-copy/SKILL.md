# Из цитат — в копирайт

Цель: тексты, собранные из формулировок клиентов. Клиент должен узнавать в тексте свои слова.

## Алгоритм
1. Карта хуков (если не строилась или банк вырос на 30%+): сгруппируй цитаты по темам, отсортируй по частоте, внутри темы выдели «формулу боли» (как чаще всего говорят) и «формулу мечты». Сохрани листом hooks в Sheets.
2. Для запрошенного текста возьми фреймворк из knowledge/COPY_FRAMEWORKS.md (PAS для холодного трафика, JTBD для лендинга, «боль → механизм → доказательство» для объявлений).
3. Правило сборки: тезисы — из топ-тем; лексика — из цитат (прямые вкрапления в кавычках или без, но с сохранением словаря клиента); обещания — только подтверждённые пользователем.
4. На каждый блок — метка происхождения: «построено на 14 цитатах темы "доставка"» (в комментарии дока, не в тексте).
5. Углы отстройки: возьми частотные боли из отзывов конкурентов, которых нет в отзывах пользователя → «у них болит X, мы говорим: у нас X решено вот так» (без имени конкурента в тексте).
6. Выход — в Google Docs «Копирайт: <дата/тема>»: варианты (заголовков — 10, блоков — 2–3 версии), у каждого пометка темы-источника.

## Краевые случаи
- В банке нет цитат по запрошенной теме → скажи прямо, предложи домайнить тему или писать без VoC (с пометкой «не подтверждено голосом клиента»).
- Пользователь просит «сделай красивее» против частотной формулировки → покажи обе версии и цифру частоты, решает он.
