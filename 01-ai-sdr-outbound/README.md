# Outbound-машина (AI SDR) / Outbound Machine

Персональный SDR: сам ресёрчит каждого проспекта, пишет не-шаблонные письма и ведёт цепочки касаний — вы только жмёте «отправить».

## Ценность и ROI

SDR в штате стоит $800–2000/мес, сервисы типа Apollo/Instantly — $100–500/мес. Ручной ресёрч одного проспекта — 20–30 минут; агент делает 20–30 в день. При подписке $29/мес окупаемость 30–70x: одна закрытая сделка с аутбаунда покрывает годы подписки.

## Что делает

1. Ведёт список проспектов в Google Sheets (добавляйте компании вручную или импортом CSV).
2. По каждому проспекту собирает досье через веб: сайт, ниша, размер, свежие новости, вакансии (сигналы боли), стек.
3. Пишет персональный опенер, привязанный к конкретному сигналу («увидел, что вы открыли вакансию X — обычно это значит Y»), без AI-штампов.
4. Строит цепочку из 3–5 касаний с интервалами; каждое утро — батч готовых черновиков в Gmail на утверждение.
5. Отслеживает ответы (триггер Gmail): классифицирует (интерес / возражение / отказ / не туда), предлагает ответ, обновляет статус в Sheets.
6. Еженедельная аналитика: отправлено, ответы, конверсия по сегментам, что работает в опенерах.

## Состав папки

- `workspace/` — копируется в воркспейс агента ASCN (IDENTITY, SOUL, ONBOARDING, скиллы, шаблоны).
- `DEPLOY_PROMPT.md` — промпт для Claude Code, который переносит всё в ASCN.
- `INTEGRATIONS.md` — справка по интеграциям. Тексты cron-задач — в workspace/AUTOMATIONS.md.

## Быстрый деплой

1. Откройте Claude Code в этой папке.
2. Вставьте содержимое `DEPLOY_PROMPT.md`.
3. Напишите агенту «Привет, начнём настройку» — дальше он сам проведёт онбординг (интеграции, ICP, тон писем, тестовый прогон).

## Требуемые интеграции

Gmail (обязательно), Google Sheets (обязательно). Канал: Telegram (рекомендуется).

## Ограничения (сознательные)

- Без LinkedIn-скрейпинга и браузерной автоматизации — ресёрч только по открытому вебу (ограничение платформы).
- Без интеграции с сендерами (Instantly/Smartlead) — отправка через Gmail с разносом по времени; при объёмах >50 писем/день рекомендуем отдельный домен и прогрев.
- Агент не «верифицирует» контакты из брокерских баз: email берутся из данных пользователя или открытых источников. Обещания «проверенных телефонов» в оффере запрещены — измеренно не работает (lib.cybos.ai, −8pp).
