# Схема реестра клиентов (Sheets «Реестр клиентов»)

## Лист clients
| Колонка | Что |
|---|---|
| client | название |
| status | active / paused / archived |
| gsc_domain | домен, как он значится в GSC (сверять точно) |
| drive_folder | ссылка на папку клиента |
| email / cc | контакт для отчёта, копия |
| blocks | что входит в отчёт: gsc, works, ads, social (через запятую) |
| notes | особенности: «без графиков», «спрашивает про лиды», «созвон после отчёта» |
| last_report / last_sent | даты, ссылка на последний отчёт |
| flag | ▲/●/▼ последнего месяца |

## Лист works (единый, фильтр по клиенту)
date | client | work_type (контент/техничка/ссылки/реклама/другое) | what | comment

## Листы ads_data / social_data (по клиентам, если включены блоки)
month | client | spend | leads | cpl | notes — формат выгрузки согласуется на онбординге; агент читает как есть, не пересчитывает чужие метрики без данных.

## Правила
Новый клиент = строка в clients + папка в Drive + проверка домена в GSC (агент делает по команде «добавь клиента …»). Изоляция: домен и папка сверяются перед каждой сборкой.
