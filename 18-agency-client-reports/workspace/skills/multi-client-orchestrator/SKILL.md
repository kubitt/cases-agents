# Оркестратор батча

> Скилл — одна повторяемая задача, описанная как рецепт. Шаги, лимиты и форматы можно править обычными словами.

Цель: все клиенты обработаны, ничего не перепутано, пользователь утверждает за 10 минут.

## Алгоритм
1. Возьми реестр: клиенты status=active. Составь план батча (порядок: сначала клиенты с флагом «сложный месяц» из напоминания о выгрузках — им нужно больше внимания пользователя).
2. По каждому — скилл client-report. После каждого: обнови реестр (report_ready + ссылка). Ошибка по клиенту (нет доступа, нет данных) — зафиксируй в реестре (error + причина), продолжай батч, не останавливайся.
3. Сводка пользователю (Telegram): «Батч <месяц>: N отчётов готово, M с проблемами». Строка на клиента: имя — итог месяца одной фразой — флаг (▲ рост / ● норма / ▼ просадка) — ссылка. Просадки первыми. Проблемные — отдельным блоком с причинами.
4. Спроси: «Готовить письма? (всем / кроме №… / только №…)».
5. Письма: черновик Gmail каждому — персональное вступление (главный итог ЕГО месяца, 2–3 фразы), ссылка/вложение отчёта, приглашение обсудить. НЕ одинаковые тексты: вступление всегда из данных клиента.
6. После финального «ок» — отправка, реестр: sent + дата. Ответы клиентов (если триггер включён): вопрос/недовольство → алерт с черновиком ответа.

## Внеочередной режим
«Отчёты всем к <дата>» — тот же цикл вне расписания. «Отчёт по <клиент>» — одиночный прогон без батч-сводки.

## Краевые случаи
- Батч прерван (лимиты, обрыв) → статусы в реестре позволяют продолжить с места остановки командой «продолжи батч»; уже готовые отчёты не пересобирай.
- У клиента 2 контакта → письмо первому, второй в cc (из реестра, колонка cc).
- Клиент на паузе (status=paused) → пропуск с пометкой в сводке.
