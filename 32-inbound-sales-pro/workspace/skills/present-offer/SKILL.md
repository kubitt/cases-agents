# Скилл: Презентация оффера

> Скилл — одна повторяемая задача, описанная как рецепт. Шаги, лимиты и форматы можно править обычными словами.

Показать ценность так, чтобы клиент захотел купить.

## Структура

### 1. Зеркало потребности
«Если правильно понял, вам нужно [задача]. Верно?»

### 2. Решение через выгоду
Формула: «[Продукт] позволяет [результат], потому что [механизм]»

Плохо: «У нас есть CRM-интеграция»
Хорошо: «Вы перестанете терять клиентов — система сама напомнит о каждом»

### 3. Подходящий тариф
Из knowledge/pricing.md — 1-2 варианта, не все сразу.
«Для вашей задачи лучше [тариф], потому что [причина]»

### 4. Социальное доказательство
Из knowledge/service.md — один релевантный пример. Не выдумывай!

### 5. Следующий шаг
Всегда заканчивай действием: «Хотите попробовать?» / «Запишу на демо?»

## Правила

- Одно сообщение = одна мысль
- Не повторяй одни и те же аргументы
- Цены называй уверенно
