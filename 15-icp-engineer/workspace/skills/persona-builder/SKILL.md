# Персоны для аутбаунда

Цель: по каждому сегменту ядра — рабочая персона, с которой SDR может писать письма уже сегодня. Не маркетинговые аватары «Мария, 34 года».

## Алгоритм
1. Для сегмента ядра возьми из данных: типовая роль ЛПР (из колонки contact_role), средний чек, цикл, источник прихода.
2. Боли: из notes выигранных сделок сегмента — какими словами клиенты описывали проблему (цитаты). Дополни веб-поиском (отзывы на конкурентов, обсуждения ниши) — с пометкой «внешний источник».
3. Триггеры покупки: события в notes перед сделкой (рост, переезд, найм, провал предыдущего решения). Каждый триггер = потенциальный сигнал для аутбаунда.
4. Возражения: из notes проигранных и затянувшихся сделок + loss_reason. К каждому — как отвечали в выигранных (если видно).
5. Скоринг проспекта по персоне: чек-лист 5–7 бинарных признаков (ниша, размер, роль, триггер, стек…), порог «горячий ≥4».
6. Запиши в ICP-документ, раздел «Персоны». Данных для пункта нет → «гипотеза, проверить интервью», не выдумывай.

## Формат персоны
Сегмент → Роль ЛПР → Боли (цитатами) → Триггеры → Возражения и ответы → Чек-лист скоринга → Где искать (площадки, источники из данных).

## Краевые случаи
- notes пустые → персона только из структурных полей + внешних источников, весь раздел с пометкой о качестве данных; порекомендуй заполнять notes в CRM.
- Персоны двух сегментов совпали → объедини сегменты в один, отметь в changelog.
