# Win/Loss анализ

Цель: найти признаки, по которым выигрыши статистически отличаются от проигрышей, — без самообмана на малых выборках.

## Алгоритм
1. Данные: Sheets «Win-Loss данные» после чистки. База: N выигрышей, M проигрышей, общая конверсия.
2. По каждому измерению (ниша, размер, источник, гео, чек, цикл сделки) посчитай: win rate среза vs средний, размер среза. Формат: «ниша X: 8/11 = 73% (среднее 22%), N=11».
3. Ранжируй сигналы по силе: |win rate среза − средний| × вес выборки. Срезы N<5 — в отдельный блок «мало данных».
4. Комбинации топ-сигналов (максимум пары): «ниша X + источник рефералы» — только если N пары ≥ 5.
5. Экономика: средний чек и цикл сделки для топ-сегментов vs анти-сегментов; посчитай «стоимость» анти-профиля (доля времени на проигрываемые сегменты).
6. Причины проигрышей: сгруппируй loss_reason по категориям (цена / не дозрели / конкурент / функционал), топ-3 с примерами формулировок из notes.
7. Запиши расчёты листом «Анализ <дата>» в Sheets, выводы — в ICP-документ (скилл persona-builder добавит персоны).

## Выход (структура вывода в чат)
Главный вывод (1 фраза) → ядро ICP (2–3 сегмента с цифрами) → анти-профиль (с ценой) → топ-3 причины проигрышей → что проверить интервью (гипотезы с малых выборок).

## Краевые случаи
- < 20 сделок всего → работай, но каждый вывод с пометкой «предварительно, N=…»; предложи дозаполнить историю.
- Один сегмент = 80% данных → сравнивать не с чем, скажи прямо и анализируй внутри сегмента (по размеру/источнику).
- Нет проигрышей в данных («записываем только победы») → анализ смещён, потребуй хотя бы список проигранных за квартал.
