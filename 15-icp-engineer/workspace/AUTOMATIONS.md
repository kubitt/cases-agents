# Автоматизации: ICP-инженер из win/loss данных

> Файл живёт в воркспейсе агента. Агент читает его при создании и изменении cron-задач. Расписания и формулировки можно менять — агент будет следовать обновлённой версии.

## Cron-задачи

### 1. Ежемесячная проверка пополнения и переанализ (ОПЦИОНАЛЬНО — создаётся только с согласия пользователя на онбординге)
- Cron: `0 11 1 * *` (1-е число месяца, 11:00)
- Промпт задачи: «Открой Google Sheets "Win-Loss сделки". Посчитай закрытые сделки (status won/lost) и сравни с last_analyzed_count из памяти. Новых ≥10 → выполни скилл winloss-analysis заново, обнови knowledge/ICP.md и Docs-копию (если подключён), пришли дельту: какие выводы усилились, какие ослабли, что появилось нового; ≤15 строк. Новых 1–9 → одно сообщение: "накопилось N новых сделок; для надёжного переанализа лучше ≥10 — запустить сейчас или подождать?" Новых 0 → ничего не отправляй.»
- Канал: Telegram (или чат ASCN).
- Примечание: для разового проекта без пополнения данных задачу не создавать — экономия кредитов. Переанализ всегда доступен командой «обнови анализ».

## Триггеры

Триггеров нет — сознательно: таблица сделок пополняется редко и батчами, событийная реакция на каждое изменение файла жгла бы кредиты без пользы. Пополнили данные — скажите «обнови анализ» или дождитесь ежемесячной проверки.

Тихие часы 23:00–08:00: задача в 11:00 вне окна.
