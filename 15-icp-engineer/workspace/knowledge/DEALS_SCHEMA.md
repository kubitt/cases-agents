# Схема выгрузки сделок

Минимум 20–30 закрытых сделок. Чем полнее notes — тем живее персоны.

## Обязательные колонки
| Колонка | Тип | Пример |
|---|---|---|
| deal_name | текст | «Внедрение для Дента-Люкс» |
| outcome | won / lost | won |
| industry | текст (нормализуется агентом) | стоматология |
| company_size | число сотрудников или диапазон | 10–50 |
| source | откуда пришли | реферал / входящая заявка / аутбаунд / реклама |
| amount | число, валюта одна на таблицу | 250000 |
| date_opened / date_closed | даты | 2026-01-10 / 2026-03-02 |

## Желательные колонки
| Колонка | Зачем |
|---|---|
| contact_role | кто был ЛПР — основа персон |
| loss_reason | причина проигрыша своими словами |
| notes | любые заметки: что спрашивали, что зацепило, на чём буксовало |
| competitor | с кем сравнивали |
| geo | город/регион |

## Правила
Одна строка = одна сделка (не контакт). Открытые сделки не включать (или колонка status — агент отфильтрует). Валюта одна; если разные — колонка currency, агент сконвертирует и пометит курс.
