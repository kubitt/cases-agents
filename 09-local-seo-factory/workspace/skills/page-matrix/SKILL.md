# Матрица страниц

Цель: полный список будущих страниц с честными приоритетами — что писать первым, чтобы быстрее пошёл трафик.

## Алгоритм
1. Построй декартово произведение: услуги (из USER.md) × локации. Исключи бессмысленные пары (услуга недоступна в локации — спроси пользователя при сомнении).
2. Приоритет каждой ячейки (1–5) по трём сигналам веб-поиска: есть ли спрос на «<услуга> <район>» (подсказки поиска, наличие конкурентных страниц), сила конкуренции (топ занят агрегаторами = сложнее), близость к деньгам (коммерческая услуга выше информационной).
3. Записывай в Sheets «Матрица страниц»: service, location, priority, search_volume_note (одна фраза почему такой приоритет), status=planned, url (пусто).
4. При подключённой GSC: раз в месяц сверяй — страницы со статусом published, получившие показы, повышают приоритет соседних ячеек того же района/услуги.

## Выход
Таблица + сообщение: всего ячеек, топ-10 приоритетных, рекомендуемый порядок батчей.

## Краевые случаи
- Матрица > 200 ячеек → предложи сузить (топ-районы или топ-услуги), не генерировать хвост с priority=1.
- Дубли локаций (Западный/ЗАО) → нормализуй названия, спроси предпочтительный вариант написания.
