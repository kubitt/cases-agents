# Генератор страниц

Цель: страница, которую нельзя получить заменой названия города в шаблоне. Уникальность — из реальных локальных деталей.

## Алгоритм
1. Возьми структуру из knowledge/PAGE_TEMPLATE.md (или образец пользователя, если задан).
2. Собери локальные детали по методике knowledge/LOCAL_SIGNALS.md: 3–5 реальных фактов о районе, релевантных услуге (типы застройки, ориентиры, специфика). Каждый факт — проверен веб-поиском. Не нашёл деталей — генери без них, но пометь страницу в матрице flag=generic (не выдумывай «уютные дворики»).
3. Напиши страницу: H1 «<Услуга> в <локации>»; лид с прямым ответом (что, где, сколько стоит от); блок «Особенности района» на локальных деталях; услуги/цены; FAQ 4–6 вопросов (реальные формулировки из поиска «<услуга> <район>»); CTA с телефоном/формой.
4. Добавь технический блок в конце документа: title (≤60 зн.), meta description (≤155 зн.), JSON-LD schema LocalBusiness + FAQPage — готовым кодом для вставки.
5. Объём 400–700 слов. Тон — из образца пользователя. Запрещено: «лучшая компания», враньё про адрес офиса в районе, если его там нет, накрутка ключей.
6. Сохрани в Google Docs (папка батча в Drive), обнови матрицу: status=draft→ready после утверждения, url после публикации.

## Батч-режим (5 страниц)
Сначала план: список 5 ячеек по приоритету — на утверждение. После «ок» — генерация, папка «SEO-страницы/Батч N от <дата>», внутри страницы + INSTRUCTIONS.md (как опубликовать: куда вставить title/meta/schema, чек-лист перелинковки на соседние страницы услуг).

## Краевые случаи
- Две соседние страницы получаются похожими → перепиши вторую с другим углом (другие FAQ, другие детали), похожесть — главный риск фабрики.
- Пользователь публикует с изменениями → при простановке url прочитай опубликованную версию и внеси стилевые отличия в PAGE_TEMPLATE.md.
