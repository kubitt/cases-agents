# Цепочка касаний

> Скилл — одна повторяемая задача, описанная как рецепт. Шаги, лимиты и форматы можно править обычными словами.

Цель: 3–5 писем, которые читаются как письмо занятого человека занятому человеку. Каждое следующее — короче.

## Структура цепочки (дефолт)
- T1 (день 0): опенер по сигналу. ≤120 слов. Формула: наблюдение о НИХ (1–2 фразы, из досье) → мостик к боли → чем помогаем таким компаниям (1 фраза, без фичей) → CTA-вопрос («актуально обсудить?» / «кому у вас переслать?»).
- T2 (день 3): новый угол — кейс/цифра по нише из ICP_PLAYBOOK.md. ≤80 слов. Не «просто напоминаю».
- T3 (день 7): полезность без просьбы (наблюдение, идея, ссылка на их же ситуацию). ≤60 слов.
- T4 (день 12, опционально): break-up. ≤40 слов: «Закрываю тему. Если станет актуально — вы знаете, где я. Успехов с <конкретное из досье>».
- Тема письма: 3–5 слов, строчными, про НИХ («филиал в казани», «онлайн-запись на dentalux»). Запрещены: капс, «выгодное предложение», emoji, «Re:» без реальной переписки.

## Правила текста
- Тон и подпись — из knowledge/VOICE.md. Пиши так, как пользователь говорит, а не «как маркетолог».
- Запрещённые обороты: «Надеюсь, у вас всё хорошо», «Меня зовут… я представляю…» в первой строке, «уникальное решение», «взаимовыгодное сотрудничество», любые плейсхолдеры «[Имя]» в готовом черновике.
- Один CTA на письмо. Вопрос, на который легко ответить одним словом.
- Каждый факт — из досье с источником. Факта нет → перепиши без него.

## Выход
Черновик в Gmail (кому, тема, тело, подпись). В Sheets: status=in_sequence, next_touch=дата, opener_type (signal_news / signal_vacancy / signal_site / case_number / breakup) — по нему пятничная аналитика считает, что работает.

## Краевые случаи
- Ответ пришёл в любом месте цепочки → цепочка стоп, дальше ручной режим с черновиками ответов.
- Автоответ «в отпуске до N» → next_touch = N+1, счётчик касаний не увеличивай.
- Отказ/негатив → do_not_contact, вежливый финальный ответ только по запросу пользователя.
