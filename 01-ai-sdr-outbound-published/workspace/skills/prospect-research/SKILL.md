# Ресёрч проспекта

> Скилл — одна повторяемая задача, описанная как рецепт. Шаги, лимиты и форматы можно править обычными словами.

Цель: за 5–10 минут собрать досье, дающее ОДИН конкретный сигнал для персонального опенера.

## Алгоритм
1. Прочитай сайт компании: главная, «о нас», услуги/продукты, цены. Зафиксируй: что продают, кому, масштаб (команда, города, клиенты).
2. Веб-поиск: «<компания> новости», «<компания> отзывы», за последние 6 месяцев. Ищи события: запуск продукта, раунд, переезд, новые филиалы, смена руководства.
3. Вакансии компании (поиск «<компания> вакансии»): открытые роли = сигналы боли. Нанимают продавцов → растят выручку; нанимают саппорт → тонут в клиентах.
4. ЛПР: если контакт не задан, найди на сайте/в открытых источниках имя владельца или профильного руководителя. Email бери только с официальных страниц; не нашёл — пометь need_contact, НЕ выдумывай адрес.
5. Выбери главный сигнал — самый свежий и наиболее связанный с оффером пользователя (ICP_PLAYBOOK.md). Сформулируй мостик: «сигнал → их вероятная боль → наш оффер».

## Выход (в Sheets, колонки signal/notes; полное досье — при запросе)
```
Компания: Дента-Люкс | dentalux.ru | стоматология, 3 клиники, ~50 сотрудников
Сигнал: 2 недели назад открыли филиал в Казани + вакансия администратора (hh, 05.07)
Мостик: новый филиал → поток первичных обращений → теряют заявки без CRM
ЛПР: Ирина Соколова, управляющая (источник: dentalux.ru/team)
Email: info@dentalux.ru (общий; личный не найден)
Источники: [3 URL]
```

## Краевые случаи
- Сайта нет / не открывается → статус need_info, спроси пользователя, не трать больше 3 минут.
- Сигналов нет (тихая компания) → сигнал = факт из их же сайта («у вас на сайте 14 услуг, но онлайн-записи нет»); честность важнее яркости.
- Дубликат (компания уже в таблице) → не создавай строку, обнови существующую и отметь duplicate_check в notes.
